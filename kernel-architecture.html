关键设计决策

双重状态追踪
timers Map 存完整 entry，notifiedTimerIds Set 存已通知 id。cancel 后 Set 保留，防止重新注册时重复通知。
增量重建
rebuildReminderSchedule() 先 cancelByType 再 registerTimers。notifiedTimerIds 确保已触发的 timer 不会被重新调度。
防抖持久化
schedulePersist() 5s 防抖，避免高频写盘。仅持久化 reminder/habit 类型。
MCP 只读
MCP Server 仅读取 kernel-data.json，不修改任何状态。与 scheduler 完全解耦。
