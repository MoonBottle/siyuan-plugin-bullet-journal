|---------|------|
| **项目背景** | 为思源笔记开发一款子弹笔记风格的任务管理插件 |

## 视图使用场景

### 日历视图 - 每日开发安排

**使用场景**：每天早上打开日历视图，查看今天的开发任务。

**操作步骤**：
1. 点击顶栏 📅 日历图标
2. 切换到「日视图」查看今天的事项
3. 点击任意事项可直接跳转到笔记对应位置

**示例**：
- 1月15日：月视图样式调整
- 1月21日：视图切换功能（已完成）
- 2月14日：使用案例编写（已完成）

### 甘特图 - 功能模块时间线

**使用场景**：查看各功能模块的开发进度和时间安排。

**操作步骤**：
1. 点击顶栏 📊 甘特图图标
2. 查看任务层级关系（L1任务展开显示L2子任务）
3. 了解各功能模块的时间跨度

**示例**：
```
第一阶段：基础架构    [1月6日 - 1月13日]
├── 搭建插件框架      [1月6日 - 1月8日] ✓
└── 数据解析器开发    [1月9日 - 1月13日]

第二阶段：日历视图    [1月14日 - 1月21日]
├── 月视图实现        [1月14日 - 1月16日] ✓
└── 周视图和日视图    [1月17日 - 1月21日] ✓
```

### 待办 Dock - 快速查看今日待办

**使用场景**：开发过程中快速查看和处理待办事项。

**操作步骤**：
1. 右键顶栏 → 关闭的面板 → 待办事项
2. 在侧边栏查看「今天」分组的事项
3. 悬停事项显示操作按钮（完成、迁移、放弃）

**分组展示**：
- **今天**：今日待办事项
- **明天**：明日计划
- **未来**：后续安排
- **已完成**：已完成的事项（可隐藏）
- **已放弃**：放弃的事项（可隐藏）
- **已过期**：逾期未处理的事项（红色标记）

## 研发工作流

### 每日工作流（5分钟）

```
1. 打开待办 Dock
   └── 查看「今天」分组的事项

2. 处理当日事项
   ├── 完成 → 点击 ✓ 标记为已完成
   ├── 需要延期 → 点击 → 迁移到明天
   └── 不再做 → 点击 ✗ 标记为已放弃

3. 查看「已过期」
   └── 处理逾期未完成的事项
```

### 每周工作流（15分钟）

```
1. 打开日历视图（周视图）
   └── 回顾本周完成的任务

2. 规划下周工作
   ├── 在笔记中添加下周的任务事项
   ├── 使用 @日期 标记具体日期
   └── 合理分配工作量

3. 检查项目进度
   └── 打开甘特图查看整体进度
```

### 每月工作流（30分钟）

```
1. 甘特图回顾
   ├── 查看各阶段完成情况
   ├── 识别延期任务
   └── 调整后续计划

2. 项目文档整理
   ├── 更新项目描述
   ├── 添加新的任务计划
   └── 归档已完成的内容

3. 下月规划
   └── 在甘特图中规划下月里程碑
```

## 最佳实践

### 1. 任务粒度控制

**建议的任务粒度**：
- ✅ **合适**：「月视图样式调整」（1-2天）
- ❌ **太大**：「日历视图开发」（跨度太长）
- ❌ **太小**：「修改按钮颜色」（太细碎）

**任务层级使用**：
- **@L1**：大功能模块（如「日历视图开发」）
- **@L2**：具体任务（如「月视图实现」）
- **@L3**：可选，用于更细分的子任务

### 2. 日期标记规范

**基本日期**：
```markdown
任务名称 #任务 @L2
事项描述 @2026-01-15
```

**带时间范围**（适用于会议、集中开发时段）：
```markdown
代码评审 #任务 @L2
评审会议 @2026-01-15 14:00~16:00
```

**多天事项**（同一任务跨多天）：
```markdown
功能开发 #任务 @L2
第一天工作 @2026-01-15
第二天工作 @2026-01-16
第三天工作 @2026-01-17
```

### 3. 状态流转流程

```
当日待办（无标签）
    │
    ├─→ 完成 → 添加 #已完成
    │           └── 显示删除线，可隐藏
    │
    ├─→ 延期 → 修改 @日期 为明天或未来
    │           └── 事项自动迁移到新日期
    │
    └─→ 放弃 → 添加 #已放弃
                └── 显示删除线+灰色，可隐藏
```

**状态标记时机**：
- **#已完成**：任务确实完成，代码已提交
- **#已放弃**：需求变更或不再做，记录决策
- **迁移日期**：任务延期，保持跟踪

### 5. 文档维护建议

**每日维护**（1分钟）：
- 标记已完成的事项
- 迁移未完成的事项

**每周维护**（5分钟）：
- 添加下周的任务计划
- 调整不合理的日期安排

**每月维护**（10分钟）：
- 归档已完成的大任务
- 更新项目整体进度描述
- 添加新的里程碑计划

## 总结

通过以上案例，可以看到子弹笔记插件在研发管理中的核心价值：

1. **记录驱动**：专注于记录而非提醒，适合开发者的自我驱动工作模式
2. **分散书写，集中查看**：在笔记中随时记录想法，通过视图集中管理
3. **视图即回顾**：日历看今日、甘特图看进度、Dock看待办
4. **双向链接**：点击任意任务跳转到笔记，保持上下文

希望本案例能帮助你更好地使用子弹笔记插件管理自己的研发项目！
