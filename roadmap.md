## 笔记格式约定

### 任务 + 事项结构
```markdown
# 项目名称

## L1 任务标题 @L1 @2024-01-15   ← Task（任务）
  - 购买办公用品 @今天              ← Item（待办）
  - 提交周报 @2024-01-15 14:00~16:00 ← Item（带时间=事件）
  - 完成报告 @今天 #done           ← Item（已完成）
  - 整理需求 @今天 #abandoned     ← Item（放弃）

## L2 子任务 @L2 @2024-01-16
  - 链接文档 [[需求文档]]           ← 任务链接（原有功能）
  - 备注：需要提前准备材料           ← 任务详情（原有功能）
```

### 事项处理流程
```
当日事项
    │
    ├─→ 完成 → 加 #done 标签
    │
    ├─→ 迁移 → 改 @日期（明天或未来）
    │
    └─→ 放弃 → 加 #abandoned 标签
```

### 标签说明
| 标签 | 含义 | 状态 | UI 效果 |
|------|------|------|---------|
| 无（必须带 @日期） | 待办 | pending | 正常显示 |
| `#done` | 已完成 | completed | 删除线 |
| `#abandoned` | 放弃 | abandoned | 删除线 + 灰色 |

### 迁移操作
```markdown
- 整理需求 @今天              ← 原始
- 整理需求 @2024-01-16        ← 迁移到明天（只改日期）
```
