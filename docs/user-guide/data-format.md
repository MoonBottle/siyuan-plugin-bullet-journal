# 数据格式

本文档详细介绍子弹笔记插件的任务和事项标记语法。

## 文档结构

项目文档分为三个主要区域，每个区域解析不同类型的信息：

### 1. 项目信息区（文档顶部）

位置：任务列表之前的内容

| 元素 | 格式 | 说明 |
|------|------|------|
| 项目名称 | `## 项目名称` | 文档标题 |
| 项目描述 | `> 项目描述` | 项目简述 |
| 项目链接 | `[链接名称](URL)` | 支持多个链接 |

### 2. 任务区

位置：包含 `#任务` 标记的内容

| 元素 | 格式 | 说明 |
|------|------|------|
| 任务标记 | `#任务` | 标识这是任务行 |
| 层级 | `@L1/@L2/@L3` | L1 一级任务，L2/L3 子任务 |
| 任务链接 | `[链接名称](URL)` | 关联文档或外部链接（单独一行） |

> ⚠️ 注意：任务本身不挂时间，时间都放在事项上

### 3. 事项区

位置：任务下方的内容（包含 `@` 但不含 `#任务`）

| 元素 | 格式 | 说明 |
|------|------|------|
| 工作事项 | 任意文本 + `@日期` | 事项内容 |
| 日期 | `@YYYY-MM-DD` | 事项日期 |
| 时间范围 | `@YYYY-MM-DD HH:mm:ss~HH:mm:ss` | 事项时间范围 |
| 多日期 | `@日期1, 日期2~日期3` | 同一事项多个日期 |
| 事项链接 | `[链接名称](URL)` | 单独一行，写在事项下方（缩进可选） |

## 标记说明

| 标记 | 说明 | 示例 |
|------|------|------|
| `##` | 项目名称 | `## 订单系统开发` |
| `>` | 项目描述 | `> 优化下单流程，提升转化率` |
| `#任务` | 标识为任务行 | `开发登录模块 #任务` |
| `@L1/@L2/@L3` | 任务层级 | `@L1` 一级任务，`@L2` 子任务 |
| `@YYYY-MM-DD` | 日期标记 | `@2026-02-25` |
| `@YYYY-MM-DD HH:mm:ss~HH:mm:ss` | 时间范围 | `@2026-02-25 14:00:00~16:00:00` |
| `[名称](URL)` | 链接 | `[需求文档](siyuan://blocks/202602...)` |

## 状态标签

事项可以通过标签标记状态，支持中英文：

| 状态 | 中文标签 | 英文标签 | 说明 |
|------|---------|---------|------|
| 待办 | 无（默认） | 无（默认） | 正常显示 |
| 已完成 | `#已完成` | `#done` | 显示删除线 |
| 已放弃 | `#已放弃` | `#abandoned` | 显示删除线 + 灰色 |

**自动识别**：解析时同时支持中英文标签，会根据系统语言自动选择打标语言。

## 多日期事项

一个事项可以关联多个日期，适用于需要在多天执行同一任务的场景。

### 基本语法

```markdown
事项名称 @2026-03-06, 2026-03-10, 2026-03-15
```

### 日期范围

使用 `~` 表示连续的日期范围：

```markdown
事项名称 @2026-03-10~2026-03-12
```

简写格式（同月内）：

```markdown
事项名称 @2026-03-10~03-12
```

### 带时间的多日期

每个日期可以单独设置时间：

```markdown
整理资料 @2026-03-06 09:00:00~09:30:00, 2026-03-10 14:00:00~15:00:00
```

日期范围可以共享相同时间：

```markdown
整理资料 @2026-03-10~03-12 14:00:00~15:00:00
```

### 混合模式

支持任意组合：

```markdown
整理某些资料 @2026-03-06 09:00:00~09:30:00, 2026-03-10~03-12 14:00:00~15:00:00, 2026-03-15 10:00:00~11:00:00
```

这表示：
- 3月6日 09:00-09:30
- 3月10日、11日、12日 每天 14:00-15:00
- 3月15日 10:00-11:00

### 多日期在日历中的显示

多日期事项会在日历的每个对应日期都显示一个事件。拖动任一事件可以：

- **修改日期**：将该日期移动到新日期，其他日期保持不变
- **添加时间**：为特定日期添加或修改时间
- **自动优化**：系统会自动合并连续日期为范围，拆分不连续日期

**示例**：

原始：`整理资料 @2026-03-06, 2026-03-10~03-12`

拖动 3月11日 到 3月20日（范围外）：`整理资料 @2026-03-06, 2026-03-10, 2026-03-12, 2026-03-20`

拖动 3月10日 到 3月07日（与3月6日连续）：`整理资料 @2026-03-06~03-07, 2026-03-11~03-12`

## 事项处理流程

```
当日事项（无标签）
    │
    ├─→ 完成 → 添加 #已完成（或 #done）
    │
    ├─→ 迁移 → 修改 @日期（明天或未来）
    │
    └─→ 放弃 → 添加 #已放弃（或 #abandoned）
```

## 任务与事项的关系

任务和事项是层级关系：

```
任务（#任务 @L1）        ← 任务本身不带日期
├── 事项 @2026-02-20    ← 事项带日期
├── 事项 @2026-02-21    ← 同一任务下可以有多个事项
└── 子任务（#任务 @L2）
    └── 事项 @2026-02-22
```

**设计原因**：
- 任务代表一个目标或交付物
- 事项代表完成这个目标的具体行动
- 一个任务可能需要多天完成，每天有不同的工作事项

## 日期格式详解

### 基本日期

```markdown
事项名称 @2026-02-25
```

### 带时间范围

```markdown
事项名称 @2026-02-25 14:00:00~16:00:00
```

> 💡 **提示**：时间范围会影响甘特图中的任务时长显示，并可在设置中配置扣除午休时间。

### 多日期

```markdown
事项名称 @2026-03-06, 2026-03-10~03-12, 2026-03-15
```

> 💡 **提示**：多日期事项会在日历的每个日期都显示，方便跟踪跨天任务。

## 链接格式

支持两种链接类型：

### 外部链接

```markdown
[需求文档](https://example.com/requirements)
[研发甘特图](https://gantt.example.com/project123)
```

### 思源内部链接

```markdown
[登录模块设计](siyuan://blocks/20260220112000)
```

> 💡 **提示**：在思源笔记中，每个块都有唯一的 blockId，可以通过 `siyuan://blocks/` 协议直接跳转。

### 事项链接

事项可以关联多个链接，链接写在事项下方：

```markdown
- 工作事项 @2026-03-10
[需求文档](https://example.com)
[设计稿](siyuan://blocks/202602...)
```

也可以缩进（可选）：

```markdown
- 工作事项 @2026-03-10
  [需求文档](https://example.com)
  [设计稿](siyuan://blocks/202602...)
```

> 💡 **提示**：事项链接写在事项下方的独立行，缩进与否都可以。多个链接可以连续写，每个链接一行。

## 常见问题

### Q: 任务和事项的区别是什么？

任务是一个目标或交付物，事项是完成这个目标的具体行动。任务不带日期，事项必须带日期。

### Q: 如何让一个任务跨越多天？

**方式一**：在任务下创建多个事项，每个事项分配不同的日期：

```markdown
开发登录模块 #任务 @L1
需求评审 @2026-02-20
编码开发 @2026-02-21
测试验收 @2026-02-22
```

**方式二**：使用多日期事项，一个事项关联多个日期：

```markdown
开发登录模块 #任务 @L1
开发工作 @2026-02-20~2026-02-22
```

### Q: 如何标记任务完成？

任务本身没有状态，状态是标记在事项上的。当所有事项都完成后，可以认为任务完成。

### Q: 支持重复任务吗？

不支持。本插件的设计理念是「记录 > 提醒」，强调记录和回顾，而非提醒驱动。

### Q: 多日期事项和多个单日期事项有什么区别？

**多日期事项**：同一内容在多天执行，如「整理资料」需要在3天分别完成。

```markdown
整理资料 @2026-03-06, 2026-03-10, 2026-03-15
```

**多个单日期事项**：每天有不同的工作内容，如「需求评审」、「编码开发」、「测试验收」。

```markdown
需求评审 @2026-03-06
编码开发 @2026-03-10
测试验收 @2026-03-15
```

选择哪种方式取决于你的记录习惯：如果每天的工作内容相同，使用多日期事项；如果每天工作内容不同，使用多个单日期事项。
