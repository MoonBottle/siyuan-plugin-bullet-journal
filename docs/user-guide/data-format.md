# 数据格式

本文档详细介绍任务助手插件的任务和事项标记语法。

## 文档结构

项目文档分为三个主要区域，每个区域解析不同类型的信息：

### 1. 项目信息区（文档顶部）

位置：任务列表之前的内容

| 元素 | 格式 | 说明 |
|------|------|------|
| 项目名称 | `## 项目名称` | 文档标题 |
| 项目描述 | `> 项目描述` | 项目简述 |
| 项目链接 | `[链接名称](URL)` | 支持多个链接 |

### 2. 任务区

位置：包含 `#任务` 标记的内容

| 元素 | 格式 | 说明 |
|------|------|------|
| 任务标记 | `#任务` 或 `📋` | 标识这是任务行 |
| 层级 | `@L1/@L2/@L3` | L1 一级任务，L2/L3 子任务 |
| 任务链接 | `[链接名称](URL)` | 关联文档或外部链接（单独一行） |

> ⚠️ 注意：任务本身不挂时间，时间都放在事项上

### 3. 事项区

位置：任务下方的内容（包含 `@` 或 `📅` 但不含 `#任务`）

| 元素 | 格式 | 说明 |
|------|------|------|
| 工作事项 | 任意文本 + `@日期` 或 `📅日期` | 事项内容 |
| 日期 | `@YYYY-MM-DD` 或 `📅YYYY-MM-DD` | 事项日期 |
| 时间范围 | `@YYYY-MM-DD HH:mm~HH:mm` / `@YYYY-MM-DD HH:mm:ss~HH:mm:ss` | 事项时间范围 |
| 多日期 | `@日期1, 日期2~日期3` | 同一事项多个日期 |
| 事项链接 | `[链接名称](URL)` | 单独一行，写在事项下方（缩进可选） |

## 标记说明

### 基础标记

| 标记 | 格式 | 说明 |
|------|------|------|
| 项目名称 | `## 项目名称` | 文档标题 |
| 项目描述 | `>` | 项目简述 |
| 任务标记 | `#任务` / `#task` / `📋` | 标识为任务行 |
| 任务层级 | `@L1/@L2/@L3` | L1 一级任务，`@L2/L3` 子任务 |
| 日期 | `@YYYY-MM-DD` / `📅YYYY-MM-DD` | 日期标记 |
| 时间范围 | `@YYYY-MM-DD HH:mm~HH:mm` / `@YYYY-MM-DD HH:mm:ss~HH:mm:ss` / `📅YYYY-MM-DD HH:mm~HH:mm` / `📅YYYY-MM-DD HH:mm:ss~HH:mm:ss` | 带时间的日期 |
| 链接 | `[名称](URL)` | 支持思源内部链接和外部链接 |

### Emoji 标记（推荐）

为了提升可读性，插件支持使用 Emoji 替代部分文本标记：

| 功能 | Emoji 标记 | 等效文本标记 |
|------|-----------|-------------|
| 任务 | `📋` | `#任务` / `#task` |
| 日期 | `📅` | `@` |
| 已完成 | `✅` | `#done` / `#已完成` |
| 已放弃 | `❌` | `#abandoned` / `#已放弃` |

**使用示例**：

```markdown
首页改版 📋 @L1

设计首页原型 📅2026-03-09

完成首页设计 📅2026-03-10 ✅

放弃旧方案 📅2026-03-08 ❌
```

## 状态标签

事项可以通过标签标记状态，支持中英文和 Emoji：

| 状态 | 中文标签 | 英文标签 | Emoji 标签 | 说明 |
|------|---------|---------|-----------|------|
| 待办 | 无（默认） | 无（默认） | 无（默认） | 正常显示 |
| 已完成 | `#已完成` | `#done` | `✅` | 显示删除线 |
| 已放弃 | `#已放弃` | `#abandoned` | `❌` | 显示删除线 + 灰色 |

**任务列表形式**：完成状态也可用任务列表（复选框）维护：行首使用 `[x]` 或 `[X]` 表示已完成，`[ ]` 表示待办。与标签可同时使用，例如 `- [x] 事项内容 @2026-03-10`。

**自动识别**：解析时同时支持中英文标签和 Emoji，会根据系统语言自动选择打标语言。

## 优先级标记

为事项设置优先级，支持 Emoji 标记：

| 优先级 | Emoji 标记 | 说明 |
|--------|-----------|------|
| 高优先级 | `🔥` | 紧急重要事项 |
| 中优先级 | `🌱` | 一般事项 |
| 低优先级 | `🍃` | 可延后处理 |

**使用示例**：

```markdown
紧急修复线上bug 📅2026-03-09 🔥

日常代码审查 📅2026-03-09 🌱

优化文档格式 📅2026-03-09 🍃
```

**交互方式**：
- **UI 面板**：事项详情 → 设置优先级
- **右键菜单**：右键点击事项 → 设置优先级
- **斜杠命令**：`/优先级`
- **手动输入**：直接编辑 Markdown，添加 Emoji 标记

## 提醒功能

为事项设置提醒时间，在指定时间通过系统通知提醒用户。

### 提醒标记

使用 `⏰` 作为提醒标记：

| 类型 | 中文格式 | 英文格式 | 说明 |
|------|----------|----------|------|
| 绝对时间 | `⏰10:00` | `⏰10:00` | 在指定日期的当天 10:00 提醒 |
| 相对开始 | `⏰提前5分钟` | `⏰5 minutes before` | 基于事项开始时间提前提醒 |
| 相对结束 | `⏰结束前30分钟` | `⏰30 minutes before end` | 基于事项结束时间提前提醒 |

### 使用示例

```markdown
周会 📅2026-03-17 ⏰14:00

周会 📅2026-03-17 14:00~16:00 ⏰提前10分钟

周会 📅2026-03-17 14:00~16:00 ⏰结束前10分钟
```

> 提示：事项时间同时支持 `HH:mm` 与 `HH:mm:ss`。在日历中拖动或拉伸事项后，插件会保留原始时间精度。

**相对提醒单位**：支持分钟、小时、天

**交互方式**：
- **UI 面板**：事项详情 → 设置提醒
- **右键菜单**：右键点击事项 → 设置提醒
- **斜杠命令**：`/提醒`
- **手动输入**：直接编辑 Markdown，添加 `⏰` 标记

## 重复事项

为事项设置重复规则，适用于周期性任务（如周会、月度汇报）。

### 重复规则标记

使用 `🔁` 作为重复标记：

| 中文格式 | 英文格式 | 说明 |
|----------|----------|------|
| `🔁每天` | `🔁daily` | 每天重复 |
| `🔁每周` | `🔁weekly` | 每周重复 |
| `🔁每周一三五` | `🔁weekly on Mon,Wed,Fri` | 每周指定周几重复 |
| `🔁每月` | `🔁monthly` | 每月重复（保持当前日号） |
| `🔁每月15日` | `🔁monthly on day 15` | 每月固定日期重复 |
| `🔁每年` | `🔁yearly` | 每年重复 |
| `🔁工作日` | `🔁workday` | 工作日重复（跳过周末） |

### 结束条件标记

| 中文格式 | 英文格式 | 说明 | 示例 |
|----------|----------|------|------|
| `截止到YYYY-MM-DD` | `until YYYY-MM-DD` | 按日期结束 | `🔁每月 截止到2026-12-31` |
| `剩余N次` | `N times remaining` | 按次数结束 | `🔁每天 剩余10次` |

### 使用示例

```markdown
月度汇报 📅2026-03-17 ⏰14:00 🔁每月 截止到2026-12-31

周会 📅2026-03-06 ⏰09:00 🔁每周 剩余52次

背单词 📅2026-03-17 ⏰08:00 🔁每天 剩余30次

日报 📅2026-03-17 ⏰17:00 🔁工作日
```

### 自动创建下次

标记为 `#done` 或 `✅` 时，系统会自动创建下次 occurrence：

```markdown
# 原事项
周会 📅2026-03-17 🔁每周

# 用户添加 ✅ 后
周会 📅2026-03-17 🔁每周 ✅

# 系统自动创建（同级插入新 block）
周会 📅2026-03-24 🔁每周
```

**交互方式**：
- **UI 面板**：事项详情 → 设置重复
- **右键菜单**：右键点击事项 → 设置重复
- **斜杠命令**：`/重复`
- **手动输入**：直接编辑 Markdown，添加 `🔁` 标记

## 多日期事项

一个事项可以关联多个日期，适用于需要在多天执行同一任务的场景。

### 基本语法

```markdown
事项名称 @2026-03-06, 2026-03-10, 2026-03-15
```

### 日期范围

使用 `~` 表示连续的日期范围：

```markdown
事项名称 @2026-03-10~2026-03-12
```

简写格式（同月内）：

```markdown
事项名称 @2026-03-10~03-12
```

### 带时间的多日期

每个日期可以单独设置时间：

```markdown
整理资料 @2026-03-06 09:00:00~09:30:00, 2026-03-10 14:00:00~15:00:00
```

也支持更常用的时分格式：

```markdown
整理资料 @2026-03-06 09:00~09:30, 2026-03-10 14:00~15:00
```

日期范围可以共享相同时间：

```markdown
整理资料 @2026-03-10~03-12 14:00:00~15:00:00
```

### 混合模式

支持任意组合：

```markdown
整理某些资料 @2026-03-06 09:00:00~09:30:00, 2026-03-10~03-12 14:00:00~15:00:00, 2026-03-15 10:00:00~11:00:00
```

这表示：
- 3月6日 09:00-09:30
- 3月10日、11日、12日 每天 14:00-15:00
- 3月15日 10:00-11:00

### 多日期在日历中的显示

多日期事项会在日历的每个对应日期都显示一个事件。拖动任一事件可以：

- **修改日期**：将该日期移动到新日期，其他日期保持不变
- **添加时间**：为特定日期添加或修改时间
- **自动优化**：系统会自动合并连续日期为范围，拆分不连续日期

**示例**：

原始：`整理资料 @2026-03-06, 2026-03-10~03-12`

拖动 3月11日 到 3月20日（范围外）：`整理资料 @2026-03-06, 2026-03-10, 2026-03-12, 2026-03-20`

拖动 3月10日 到 3月07日（与3月6日连续）：`整理资料 @2026-03-06~03-07, 2026-03-11~03-12`

## 标记顺序约定

推荐按以下顺序书写标记：

```
内容 @日期 [⏰提醒时间] [🔁重复规则] [截止至日期|剩余次数] [🔥优先级] [✅状态]
```

**完整示例**：

```markdown
月度汇报 📅2026-03-17 ⏰14:00 🔁每月 截止到2026-12-31 🔥
```

**容错规则**：
- **不强制顺序**：解析器通过正则匹配，不依赖位置
- **多语言混用**：取第一个有效标记，后面的忽略

## 事项处理流程

```
当日事项（无标签或 [ ]）
    │
    ├─→ 完成 → 添加 #已完成（或 #done，或 ✅），或勾选为 [x]
    │
    ├─→ 迁移 → 修改 @日期（明天或未来）
    │
    └─→ 放弃 → 添加 #已放弃（或 #abandoned，或 ❌）
```

## 习惯定义

习惯与任务同级，都是项目下的一级概念，通过 `🎯` 标记定义。

### 习惯标记

| 元素 | 格式 | 说明 |
|------|------|------|
| 习惯标记 | `🎯` | 标识这是习惯行（写在习惯名之后） |
| 开始日期 | `🎯YYYY-MM-DD` | 习惯开始日期（必填） |
| 坚持天数 | `坚持N天` | 持续日历天数（可选，不设则无限期） |
| 目标值+单位 | `8杯`、`5公里` | 计数型习惯的目标（可选） |
| 提醒时间 | `⏰HH:mm` | 习惯提醒时间（可选） |
| 频率规则 | `🔄每天` | 习惯打卡频率（必填） |

### 习惯类型

**二元型**（无目标值）：

```markdown
早起 🎯2026-04-01 坚持30天 🔄每天

冥想 🎯2026-04-01 🔄每天
```

**计数型**（有目标值+单位）：

```markdown
喝水 🎯2026-04-01 坚持21天 8杯 🔄每天

跑步 🎯2026-04-01 5公里 🔄每天
```

### 打卡记录

习惯下方的 `📅日期` 行解析为打卡记录：

```markdown
早起 🎯2026-04-01 坚持30天 🔄每天

早起 📅2026-04-06

早起 📅2026-04-07
```

计数型打卡记录：

```markdown
喝水 🎯2026-04-01 坚持21天 8杯 🔄每天

喝水 3/8杯 📅2026-04-07

喝水 8/8杯 📅2026-04-08
```

> 计数型格式为 `当前值/目标值+单位`，目标值在创建记录时从习惯定义快照复制。

### 习惯频率规则

使用 `🔄` 作为习惯频率标记：

| 中文格式 | 英文格式 | 说明 |
|----------|----------|------|
| `🔄每天` | `🔄daily` | 每天打卡 |
| `🔄每N天` | `🔄every N days` | 每N天打卡一次 |
| `🔄每周` | `🔄weekly` | 每周打卡一次 |
| `🔄每周N天` | `🔄N days/week` | 每周打卡N天 |
| `🔄每周一三五` | `🔄Mon,Wed,Fri` | 每周指定周几打卡 |

> **与 `🔁` 的区别**：`🔄` 是习惯专用频率标记，`🔁` 是事项重复标记。习惯打卡由用户主动触发，系统不会自动创建打卡记录。

更多习惯打卡详情请参阅 [习惯打卡](./habit-checkin.md)。

## 任务与事项的关系

任务和事项是层级关系：

```
任务（#任务 @L1）        ← 任务本身不带日期
├── 事项 @2026-02-20    ← 事项带日期
├── 事项 @2026-02-21    ← 同一任务下可以有多个事项
└── 子任务（#任务 @L2）
    └── 事项 @2026-02-22
```

**设计原因**：
- 任务代表一个目标或交付物
- 事项代表完成这个目标的具体行动
- 一个任务可能需要多天完成，每天有不同的工作事项

### 习惯与任务的上下文切换

习惯和任务在文档中按出现顺序切换上下文：

```markdown
## 我的项目

#任务 开发登录模块 @L1

设计登录页面 📅2026-04-01

早起 🎯2026-04-01 坚持30天 🔄每天

喝水 🎯2026-04-01 坚持21天 8杯 🔄每天

#任务 开发注册模块 @L1

编写注册接口 📅2026-04-02
```

- 遇到 `🎯` 切换为**习惯上下文**，下方的 `📅日期` 行解析为打卡记录
- 遇到 `#任务` 切换为**任务上下文**，下方的 `📅日期` 行解析为工作事项

## 日期格式详解

### 基本日期

```markdown
事项名称 @2026-02-25
事项名称 📅2026-02-25
```

### 带时间范围

```markdown
事项名称 @2026-02-25 14:00:00~16:00:00
事项名称 📅2026-02-25 14:00:00~16:00:00
```

> 💡 **提示**：时间范围会影响甘特图中的任务时长显示，并可在设置中配置扣除午休时间。

### 多日期

```markdown
事项名称 @2026-03-06, 2026-03-10~03-12, 2026-03-15
```

> 💡 **提示**：多日期事项会在日历的每个日期都显示，方便跟踪跨天任务。

## 番茄钟记录

专注完成后，插件会自动在对应事项块下追加一行番茄钟记录；你也可以在笔记中按以下格式手写，插件会解析并纳入统计与专注记录列表。

### 格式一（无实际时长）

```text
🍅YYYY-MM-DD HH:mm:ss~HH:mm:ss
```

- 日期、开始时间与结束时间，`~` 连接起止时间。
- 示例：`🍅2026-03-10 09:00:00~09:25:00`
- 行末可加可选描述文字。

### 格式二（带实际专注分钟数）

当专注过程中有过暂停时，可写明实际专注分钟数：

```text
🍅N,YYYY-MM-DD HH:mm:ss~HH:mm:ss
```

- `N` 为实际专注分钟数（整数），逗号支持中文或英文。
- 示例：`🍅20,2026-03-10 09:00:00~09:30:00` 表示该段时间内实际专注 20 分钟。

### 解析与关联规则

- 番茄钟块紧跟在**某个事项块下方**时，关联到该事项。
- 若在**任务块下方**且不隶属于某事项，则关联到该任务。
- 否则关联到项目。

插件据此在番茄专注 Dock 的统计与「专注记录」列表中显示归属，并支持点击记录跳转到笔记位置。详见 [番茄钟](./pomodoro.md)。

## 链接格式

支持三种链接类型：

### 外部链接

```markdown
[需求文档](https://example.com/requirements)
[研发甘特图](https://gantt.example.com/project123)
```

### 思源内部链接

```markdown
[登录模块设计](siyuan://blocks/20260220112000)
```

> 💡 **提示**：在思源笔记中，每个块都有唯一的 blockId，可以通过 `siyuan://blocks/` 协议直接跳转。

### 块引用链接

思源块引用也会被当作链接解析，分为两种写法：

1. 行内块引用：写在项目名、任务名、事项内容中
2. 单独一行块引用：写在任务或事项下方，作为附属链接

### 行内块引用

项目名、任务名、事项内容中的思源行内块引用 `((blockId '锚文本'))` 或 `((blockId "锚文本"))` 会被解析：

- **显示**：块引用替换为锚文本（无锚文本时移除），避免侧边栏显示混乱
- **链接**：自动加入 project/task/item 的 links，可点击跳转到对应块

示例：`首页((20260310210016-gkixdit '测试'))改版 #任务` → 任务名显示为「首页测试改版」，links 中增加可点击的「测试」链接

### 单独一行块引用

任务或事项下方的单独一行块引用也会被识别为链接：

```markdown
工作事项 @2026-03-10
((20260422075729-q6vs0km '需求文档'))
```

如果写在任务下方，则归属任务；如果写在事项下方，则归属事项。

### 事项链接

事项可以关联多个链接，链接写在事项下方：

```markdown
工作事项 @2026-03-10
[需求文档](https://example.com)
[设计稿](siyuan://blocks/202602...)
```

也可以缩进（可选）：

```markdown
工作事项 @2026-03-10
  [需求文档](https://example.com)
  [设计稿](siyuan://blocks/202602...)
```

> 💡 **提示**：事项链接写在事项下方的独立行，缩进与否都可以。多个链接可以连续写，每个链接一行。

### Todo Dock 中的链接展示

- Todo Dock 中会合并显示当前事项可访问的事项链接和任务链接
- 外部链接、思源链接、块引用链接会使用不同背景色区分
- 链接背景色会跟随当前思源主题变化

## 常见问题

### Q: 任务和事项的区别是什么？

任务是一个目标或交付物，事项是完成这个目标的具体行动。任务不带日期，事项必须带日期。

### Q: 如何让一个任务跨越多天？

**方式一**：在任务下创建多个事项，每个事项分配不同的日期：

```markdown
开发登录模块 #任务 @L1

需求评审 @2026-02-20

编码开发 @2026-02-21

测试验收 @2026-02-22
```

**方式二**：使用多日期事项，一个事项关联多个日期：

```markdown
开发登录模块 #任务 @L1

开发工作 @2026-02-20~2026-02-22
```

### Q: 如何标记任务完成？

任务本身没有状态，状态是标记在事项上的。当所有事项都完成后，可以认为任务完成。

### Q: 多日期事项和多个单日期事项有什么区别？

**多日期事项**：同一内容在多天执行，如「整理资料」需要在3天分别完成。

```markdown
整理资料 @2026-03-06, 2026-03-10, 2026-03-15
```

**多个单日期事项**：每天有不同的工作内容，如「需求评审」、「编码开发」、「测试验收」。

```markdown
需求评审 @2026-03-06

编码开发 @2026-03-10

测试验收 @2026-03-15
```

选择哪种方式取决于你的记录习惯：如果每天的工作内容相同，使用多日期事项；如果每天工作内容不同，使用多个单日期事项。

### Q: 提醒和重复有什么区别？

**提醒**（`⏰`）：在指定时间发送一次通知，用于提醒用户即将开始的事项。

**重复**（`🔁`）：定义事项的周期性，完成当前事项后自动创建下次 occurrence。

两者可以配合使用：

```markdown
周会 📅2026-03-17 ⏰09:00 🔁每周
```

这表示：每周一早上 9:00 提醒开会，完成本周会议后自动创建下周的会议事项。

### Q: Emoji 标记和文本标记可以混用吗？

可以。解析器会识别所有支持的标记格式，但建议在同一份文档中保持风格统一，便于阅读。

```markdown
# 推荐：统一使用 Emoji
首页改版 📋 @L1
设计首页 📅2026-03-09 🔥
完成设计 📅2026-03-10 ✅

# 或者：统一使用文本标记
首页改版 #任务 @L1
设计首页 @2026-03-09 #高优先级
完成设计 @2026-03-10 #done
```
