# 习惯打卡

任务助手插件支持习惯追踪功能，帮助您建立和维持日常习惯。习惯与任务同级，都是项目下的一级概念，通过 `🎯` 标记定义，支持二元型（做了/没做）和计数型（带目标值+单位）两种类型。

## 快速开始

### 第一步：在笔记中定义习惯

在项目文档中，使用 `🎯` 标记定义一个习惯：

```markdown
早起 🎯2026-04-01 坚持30天 🔄每天
```

### 第二步：在 Dock 中打卡

打开习惯打卡 Dock，点击习惯旁的「打卡」按钮即可记录今日打卡。

### 第三步：查看统计

点击习惯名称展开详情，查看连续天数、完成率和打卡日历。

## 习惯定义语法

### 基本格式

```markdown
习惯名 🎯开始日期 [坚持N天] [目标值+单位] [⏰提醒时间] [🔄频率规则]
```

**核心规则**：

- `🎯` + 开始日期为**必填项**——习惯必须有明确的起始点
- `🔄` 频率规则为**必填项**——习惯必须是周期性的
- `坚持N天` 表示从开始日期起的日历天数，到达后习惯结束
- 频率仅表示「多久打卡一次」，不绑定具体日期

### 二元型习惯（无目标值）

适用于「做了/没做」类习惯：

```markdown
早起 🎯2026-04-01 坚持30天 🔄每天

冥想 🎯2026-04-01 🔄每天

周报 🎯2026-04-01 坚持84天 🔄每周五
```

### 计数型习惯（有目标值+单位）

适用于需要量化追踪的习惯：

```markdown
喝水 🎯2026-04-01 坚持21天 8杯 🔄每天

跑步 🎯2026-04-01 5公里 🔄每天

阅读 🎯2026-04-01 30分钟 🔄每天

背单词 🎯2026-04-01 坚持90天 50个 🔄每天
```

**支持的单位**：次、杯、分钟、小时、公里、页、毫升，以及自定义单位。

### 带提醒的习惯

```markdown
喝水 🎯2026-04-01 坚持21天 8杯 ⏰09:00 🔄每天

早起 🎯2026-04-01 坚持30天 ⏰07:00 🔄每天
```

### 频率规则

使用 `🔄` 作为习惯频率标记：

| 中文格式 | 英文格式 | 说明 |
|----------|----------|------|
| `🔄每天` | `🔄daily` | 每天打卡 |
| `🔄每N天` | `🔄every N days` | 每N天打卡一次 |
| `🔄每周` | `🔄weekly` | 每周打卡一次 |
| `🔄每周N天` | `🔄N days/week` | 每周打卡N天 |
| `🔄每周一三五` | `🔄Mon,Wed,Fri` | 每周指定周几打卡 |

> **与 `🔁` 的区别**：`🔄` 是习惯专用频率标记，`🔁` 是事项重复标记。习惯打卡由用户主动触发，系统不会根据频率自动创建打卡记录。

### 习惯与任务的上下文切换

习惯和任务在文档中按出现顺序切换上下文：

```markdown
## 我的项目

📋 开发登录模块 @L1

设计登录页面 📅2026-04-01

早起 🎯2026-04-01 坚持30天 🔄每天

喝水 🎯2026-04-01 坚持21天 8杯 🔄每天

📋 开发注册模块 @L1

编写注册接口 📅2026-04-02
```

- 遇到 `🎯` 切换为**习惯上下文**，下方的 `📅日期` 行解析为打卡记录
- 遇到 `📋` 切换为**任务上下文**，下方的 `📅日期` 行解析为工作事项

## 打卡记录格式

打卡记录由用户在首次打卡时创建，位于习惯定义行下方。

### 二元型打卡记录

```markdown
早起 🎯2026-04-01 坚持30天 🔄每天

早起 📅2026-04-06

早起 📅2026-04-07
```

### 计数型打卡记录

```markdown
喝水 🎯2026-04-01 坚持21天 8杯 🔄每天

喝水 3/8杯 📅2026-04-07

喝水 8/8杯 📅2026-04-08
```

**计数型记录格式为 `当前值/目标值+单位`**。目标值在创建记录时从习惯定义复制，后续修改习惯目标不影响历史记录。

### 完成态推导

- **二元型**：记录存在即表示完成
- **计数型**：`currentValue >= targetValue` 即表示完成，无需额外标记

## 斜杠命令

### 习惯标记命令

| 命令 | 触发词 | 功能说明 |
|------|--------|----------|
| 创建/编辑习惯 | `/xg`, `/habit` | 非习惯行：打开创建弹框；已有习惯行：打开编辑弹框 |
| 打卡 | `/dk`, `/checkin` | 根据当前行类型执行快捷打卡 |
| 打开习惯 Dock | `/xgd`, `/habits` | 打开习惯打卡侧边栏 |

### `/xg` 创建习惯

在编辑器中输入 `/xg` 唤起创建弹框：

- 习惯名（文本输入）
- 开始日期（日期选择器，默认今天）
- 坚持天数（数字输入，可选，留空则无限期）
- 类型切换（二元型 / 计数型）
- 目标值+单位（计数型时显示）
- 提醒时间（可选）
- 频率（选择：每天 / 每N天 / 每周 / 每周N天 / 指定周几）

确认后在当前光标位置插入完整的习惯定义行：

```markdown
喝水 🎯2026-04-01 坚持21天 8杯 ⏰09:00 🔄每天
```

### `/dk` 打卡行为

| 触发行类型 | 行为 |
|-----------|------|
| 习惯定义行（今日未打卡） | 今日打卡一次：二元型直接创建记录，计数型当前值 +1 |
| 习惯定义行（今日已打卡） | 提示「今日已打卡」 |
| 习惯定义行（已归档） | 提示「习惯已归档」 |
| 今日打卡记录行（二元型） | 提示「今日已打卡」 |
| 今日打卡记录行（计数型） | 当前值 +1 |
| 历史打卡记录行 | 打开习惯 Dock，定位到该习惯详情 |

## 习惯打卡 Dock

习惯打卡以 Dock 形式存在，桌面端和移动端均支持。

### 桌面端

**访问方式**：右键顶栏 → 关闭的面板 → 习惯打卡

**第一层（习惯列表）**：

- **周日期行**：显示本周周一到周日，当天日期高亮，点击可切换查看某天的习惯
- **习惯列表**：每个习惯一行，显示名称 + 进度/状态
  - 二元型：`已打卡` / `待打卡` + 连续天数点阵
  - 计数型：`当前值/目标值+单位` + 进度点阵
- **已达标习惯**：当期已满足频率要求的习惯仍显示在列表中，标记为「本周已达标」，灰色淡化处理

**第二层（习惯详情）**：

点击习惯后展开详情面板：

- **统计卡片**：月打卡次数、总打卡次数、月完成率、当前连续/最长连续天数
- **打卡日历**：月视图，用颜色深浅表示达标程度（深绿=达标，浅绿=部分，灰=未打卡）
- **打卡日志**：列表形式，按时间倒序显示打卡记录
- **快捷操作**：计数型可点击 +1 或修改值

### 移动端

移动端打卡功能集成在待办 Dock 的底部导航中：

**底部导航栏**：`🍅番茄钟 | 🎯打卡 | [+ 创建] | ⚙更多`

- 点击「打卡」切换显示习惯列表
- 点击习惯项打开详情抽屉（打卡日历 + 日志 + 统计）

## 统计与连续天数

### 连续天数（Streak）

连续天数按习惯频率对应的周期计算：

| 频率 | Streak 单位 | 计算逻辑 |
|------|------------|----------|
| `🔄每天` | 天 | 从今天往前每天检查，达标+1，否则中断 |
| `🔄每N天` | 周期 | 从当前周期往前检查，每个N天周期内至少打卡1次则+1 |
| `🔄每周` | 周 | 从本周往前检查，每个自然周内至少打卡1次则+1 |
| `🔄每周N天` | 周 | 从本周往前检查，每个自然周内打卡天数 ≥ N 则+1 |

**今日未打卡的特殊处理**：今天还没结束不算中断，`currentStreak` 从昨天开始算。

### 完成率

| 指标 | 计算方式 |
|------|----------|
| 总完成率 | 累计已完成要求数 / 累计应完成要求数 |
| 本周完成率 | 本周已完成要求数 / 本周应完成要求数 |
| 本月完成率 | 本月已完成要求数 / 本月应完成要求数 |

计数型的「完成」：`currentValue >= targetValue` 算完成。

### 热力图

打卡日历使用颜色深浅表示达标程度：

- **深绿** `#52c41a`：达标
- **浅绿** `#95de64`：部分完成（计数型未达标）
- **灰色** `#e8e8e8`：未打卡
- **留空**：非打卡日

## 提醒功能

习惯提醒**不依赖打卡记录**，直接从习惯定义计算。

**计算逻辑**：

1. 先判断今天是否仍在习惯有效期内（未到 `startDate` 或超过 `endDate` 不提醒）
2. 根据频率判断今天是否需要提醒：
   - `🔄每天` → 每天提醒，今日已打卡则跳过
   - `🔄每2天` → 从 startDate 起命中周期日才提醒
   - `🔄每周` → 当周未达标前每天提醒，达标后停止
   - `🔄每周N天` → 当周未达标前每天提醒，达标后停止
   - `🔄每周一三五` → 仅指定周几提醒，当天已打卡则跳过
3. 计算提醒时间：`⏰09:00` → 今天 09:00 提醒

## 补打卡与编辑

### 补打卡

支持为历史日期补打卡：

- **Dock 中切换日期**：习惯 Dock 的周日期行支持切换查看历史日期
- **补打卡操作**：与当日打卡操作一致，创建的记录带对应历史日期
- **插入位置**：补打卡的记录按日期顺序插入到习惯定义下方的合适位置
- **streak 影响**：补打卡会影响连续天数计算

### 编辑打卡记录

- **计数型记录**：可通过 Dock UI 修改已有记录的值
- **笔记中直接编辑**：直接修改 Markdown 中的数值
- **修改后**：自动重新判断是否达标，更新 UI 完成态

### 撤销打卡

支持撤销当天或历史日期的打卡，删除对应的记录块。

## 习惯归档

当习惯不再继续时，可以通过归档将其从默认活跃列表中隐藏，同时保留历史记录和统计。

### 归档标记

在习惯定义行末尾添加 `📦YYYY-MM-DD` 标记：

```markdown
喝水 🎯2026-04-01 坚持21天 8杯 🔄每天 📦2026-05-04
```

- `📦` 表示该习惯已归档
- 后接归档日期 `YYYY-MM-DD`

### 归档与自然结束

习惯有两种非活跃状态：

| 状态 | 含义 | 触发方式 |
|------|------|----------|
| **已结束** | 自然到期 | `坚持N天` 到期后自动变为已结束 |
| **已归档** | 用户主动停用 | 手动添加 `📦` 标记 |

两者互不覆盖，可独立存在：

- 未结束但已归档
- 已结束但未归档
- 已结束且已归档

**活跃性判断**：习惯只有同时满足「未归档」且「未结束」才被视为活跃习惯。

### 归档操作

**入口**：在习惯详情面板右上角操作区执行归档或取消归档。

**归档时**：

- 弹出确认提示：「归档后将停止提醒，并从默认习惯列表中隐藏；历史记录会保留」
- 确认后在习惯定义行追加 `📦YYYY-MM-DD`

**取消归档**：

- 不弹确认，直接恢复
- 移除习惯定义行中的 `📦YYYY-MM-DD`

> 归档状态不通过「创建/编辑习惯弹框」管理，归档/取消归档走独立操作按钮。

### 归档后的行为

| 行为 | 已归档习惯 |
|------|------------|
| 默认习惯列表 | 隐藏，不出现在活跃列表中 |
| 习惯详情 | 仍可打开查看月历、统计、日志 |
| 打卡 | 禁用（按钮置灰，`/dk` 提示「习惯已归档」） |
| 提醒 | 停止触发 |
| 历史记录 | 保留，不受影响 |
| 统计 | 详情页可完整查看 |
| 文档跳转 | 可正常打开定义所在 block |

### 归档标记格式错误

若手写错误格式如 `📦今天`：

- 不识别为已归档
- 不报错中断解析
- 其余习惯字段正常解析

## 常见问题

### Q: 习惯和任务有什么区别？

**任务**是有明确交付物的目标，通过 `📋` 标记，下方的事项带日期表示具体行动。

**习惯**是周期性重复的行为追踪，通过 `🎯` 标记，下方的打卡记录表示每次执行情况。

两者在项目文档中同级存在，按出现顺序切换上下文。

### Q: 计数型习惯的目标值修改后，历史记录会变化吗？

不会。计数型打卡记录的目标值在创建时从习惯定义**快照复制**，后续修改习惯目标不影响历史记录。例如习惯从 8杯 改为 10杯 后，旧记录仍显示 `3/8杯`，新记录显示 `0/10杯`。

### Q: 习惯到期后还能补打卡吗？

可以。`坚持N天` 到期后习惯标记为已结束，但仍允许为历史日期补打卡。到期后不再生成新的提醒。

### Q: `🔄每周` 和 `🔄每周一三五` 有什么区别？

- `🔄每周`：每周内任意一天打卡一次即可，本周打卡一次后即为达标
- `🔄每周一三五`：必须在周一、周三、周五都打卡，且每周如此

### Q: 习惯打卡会影响任务和事项的解析吗？

不会。`🎯` 是新增标记，与现有 `📋`、`📅` 等标记不冲突。习惯和任务在文档中严格交替，解析器按出现顺序正确区分上下文。

### Q: 可以在同一个项目中同时有习惯和任务吗？

可以。习惯和任务在项目文档中并列存在，习惯用于追踪日常行为，任务用于管理工作目标。
