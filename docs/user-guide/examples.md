# 完整示例

本文档通过一个「电商订单系统重构」项目，展示子弹笔记插件的完整使用方式。

## 项目背景

假设您负责一个电商订单系统的重构项目，目标是优化下单流程，提升转化率 20%。

## 项目文档

在思源笔记中创建文档 `电商订单系统重构.sy.md`，内容如下：

```markdown
## 电商订单系统重构
> 优化下单流程，提升转化率 20%

[需求文档](https://example.com/requirements)
[研发甘特图](https://gantt.example.com/project123)

### 工作任务

用户登录模块开发 #任务 @L1
[登录模块设计](siyuan://blocks/20260220112000)

实现手机号登录 #任务 @L2
手机号登录开发 @2026-02-20

实现短信验证码 #任务 @L2
短信验证码开发 @2026-02-21

实现第三方登录（微信/Google）#任务 @L2
第三方登录开发 @2026-02-22

登录模块测试 @2026-02-23

购物车优化 #任务 @L1
购物车需求评审 @2026-02-24
购物车单元测试 @2026-02-25 10:00:00~12:00:00

订单结算页面开发 #任务 @L1
选择收货地址 @2026-02-26
选择支付方式 @2026-02-26
订单确认 @2026-02-27
```

## 文档结构解析

### 项目信息区

```markdown
## 电商订单系统重构
> 优化下单流程，提升转化率 20%

[需求文档](https://example.com/requirements)
[研发甘特图](https://gantt.example.com/project123)
```

- 项目名称：电商订单系统重构
- 项目描述：优化下单流程，提升转化率 20%
- 项目链接：需求文档、研发甘特图

### 任务区

项目包含三个一级任务：

1. **用户登录模块开发** - 包含三个子任务
2. **购物车优化** - 无子任务
3. **订单结算页面开发** - 无子任务

### 事项区

每个任务下有多个事项，带有日期标记：

- `手机号登录开发 @2026-02-20`
- `购物车单元测试 @2026-02-25 10:00:00~12:00:00`（带时间范围）

## 任务状态更新

### 完成事项

当事项完成后，添加 `#已完成` 标签：

```markdown
手机号登录开发 @2026-02-20 #已完成
```

![已完成事项显示效果](./images/examples-status-done.png)

### 迁移事项

如果今天没完成，修改日期迁移到明天：

```markdown
手机号登录开发 @2026-02-21
```

### 放弃事项

如果决定不做，添加 `#已放弃` 标签：

```markdown
第三方登录开发 @2026-02-22 #已放弃
```

![已放弃事项显示效果](./images/examples-status-abandoned.png)

## 最佳实践

### 1. 任务粒度

- 一级任务（@L1）：一个可交付的功能模块
- 二级任务（@L2）：模块下的具体功能点
- 事项：每天的具体工作内容

### 2. 日期规划

- 先规划任务，再分配事项日期
- 事项日期要留有缓冲
- 每天回顾并调整计划

### 3. 状态管理

- 每日回顾时更新状态
- 完成的事项及时标记
- 过期事项及时迁移或放弃

### 4. 链接使用

- 为任务添加相关文档链接
- 使用思源内部链接跳转更快
- 外部链接用于关联外部资源
