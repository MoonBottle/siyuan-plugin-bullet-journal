# 完整示例

本文档通过一个「电商订单系统重构」项目，展示任务助手插件的完整使用方式。

## 项目背景

假设您负责一个电商订单系统的重构项目，目标是优化下单流程，提升转化率 20%。

## 项目文档

在思源笔记中创建文档 `电商订单系统重构.sy.md`，内容如下：

```markdown
## 电商订单系统重构
> 优化下单流程，提升转化率 20%

[需求文档](https://example.com/requirements)
[研发甘特图](https://gantt.example.com/project123)

### 工作任务

用户登录模块开发 📋 @L1

[登录模块设计](siyuan://blocks/20260220112000)

实现手机号登录 📋 @L2

手机号登录开发 📅2026-02-20 🔥

实现短信验证码 📋 @L2

短信验证码开发 📅2026-02-21 🌱

实现第三方登录（微信/Google）📋 @L2

```

## 文档结构解析

### 项目信息区

```markdown
## 电商订单系统重构
> 优化下单流程，提升转化率 20%

[需求文档](https://example.com/requirements)
[研发甘特图](https://gantt.example.com/project123)
```

- 项目名称：电商订单系统重构
- 项目描述：优化下单流程，提升转化率 20%
- 项目链接：需求文档、研发甘特图

### 任务区

项目包含三个一级任务：

1. **用户登录模块开发** - 包含三个子任务
2. **购物车优化** - 无子任务
3. **订单结算页面开发** - 无子任务

### 事项区

每个任务下有多个事项，带有日期标记：

- `手机号登录开发 @2026-02-20`
- `购物车单元测试 @2026-02-25 10:00:00~12:00:00`（带时间范围）

## 高级功能示例

### 优先级标记

为事项设置优先级，帮助区分重要程度：

```markdown
紧急修复线上bug 📅2026-03-09 🔥

日常代码审查 📅2026-03-09 🌱

优化文档格式 📅2026-03-09 🍃
```

### 提醒设置

为重要事项设置提醒：

```markdown
周会 📅2026-03-17 14:00~16:00 ⏰13:50

项目截止 📅2026-03-20 ⏰提前1天
```

### 重复事项

周期性任务自动创建：

```markdown
每日站会 📅2026-03-17 09:00 🔁每天 截止到2026-12-31

周会 📅2026-03-17 14:00 ⏰13:50 🔁每周

月度汇报 📅2026-03-17 10:00 🔁每月15日
```

## 任务状态更新

### 完成事项

当事项完成后，添加 `#已完成` 标签或 `✅` Emoji：

```markdown
手机号登录开发 @2026-02-20 ✅
```

### 迁移事项

如果今天没完成，修改日期迁移到明天：

```markdown
手机号登录开发 @2026-02-21
```

### 放弃事项

如果决定不做，添加 `#已放弃` 标签或 `❌` Emoji：

```markdown
第三方登录开发 @2026-02-22 ❌
```

## 斜杠命令使用示例

在日常使用中，斜杠命令可以大幅提升操作效率：

### 快速标记日期

```markdown
# 输入
完成需求文档 /jt

# 结果
完成需求文档 📅2026-03-09
```

### 快速标记完成

```markdown
# 输入
修复登录 bug /wc

# 结果
修复登录 bug ✅
```

### 快速打开视图

```markdown
# 输入
周会 /rl

# 结果：打开日历视图并自动定位到周会日期
```

### 快速开始专注

```markdown
# 输入
阅读技术文档 /zz

# 结果：打开番茄钟专注弹框，预选「阅读技术文档」事项
```

## 最佳实践

### 1. 任务粒度

- 一级任务（@L1）：一个可交付的功能模块
- 二级任务（@L2）：模块下的具体功能点
- 事项：每天的具体工作内容

### 2. 日期规划

- 先规划任务，再分配事项日期
- 事项日期要留有缓冲
- 每天回顾并调整计划

### 3. 状态管理

- 每日回顾时更新状态
- 完成的事项及时标记
- 过期事项及时迁移或放弃

### 4. 链接使用

- 为任务添加相关文档链接
- 使用思源内部链接跳转更快
- 外部链接用于关联外部资源
