# 番茄钟

番茄钟功能帮助你在任务助手中进行计时专注，并将专注记录自动写入笔记，与事项一一对应。

## 访问方式

**番茄专注 Dock**：右键顶栏 → 关闭的面板 → 番茄专注

打开后，侧边栏会显示「番茄专注」面板。

## 界面说明

### 未专注时

- **统计四格**：今日番茄数、今日专注时长、总番茄数、总专注时长（基于扫描目录内所有项目文档中的番茄钟记录）。
- **专注记录列表**：按日期分组展示历史专注记录，每条显示时间范围、关联事项和时长；点击记录可跳转到笔记中对应块位置。
- **顶部操作**：播放按钮「开始专注」、刷新按钮重新加载数据。

### 专注时

- **倒计时环**：剩余时间（MM:SS）与已专注分钟数。
- **时间线**：开始时间、预计结束时间与进度条。
- **事项信息卡片**：当前专注的项目、任务、事项及关联链接；点击事项卡片可打开文档并定位到该事项。
- **操作按钮**：暂停 / 继续、结束专注（会保存当前番茄钟记录）。

## 开始专注

可从以下入口打开「开始专注」弹框：

| 入口 | 操作 |
|------|------|
| 番茄专注 Dock | 点击顶部播放图标 |
| 待办侧栏 | 点击某条事项的「开始专注」按钮 |
| 日历视图 | 点击某条事项的「开始专注」按钮 |
| 甘特图 | 点击某条事项的「开始专注」按钮 |
| 笔记中的事项块 | 右键 → 开始专注 |

弹框中：

1. **选择待办事项**：左侧列出「过期事项」与「今天事项」（均为待办状态），选择要专注的事项；若从日历/甘特图/待办/右键某事项进入，可能已预选该事项且不显示列表。
2. **设置专注时长**：快捷 15 / 25 / 45 / 60 分钟，或自定义 1–180 分钟。
3. 点击「开始专注」即开始计时；若从 Dock 外入口开始，成功后会自动切换到番茄专注 Dock。

## 专注中操作

- **暂停**：计时暂停，再次打开 Dock 可「继续」；暂停期间不计入专注时长。
- **继续**：从当前剩余时间继续计时。
- **结束专注**：确认后会将本段专注**作为一条番茄钟记录追加到当前事项块下方**，并播放提示音、弹出系统通知（若已授权）。记录格式见下文。

专注状态会持久化：关闭 Dock 或重启思源后，再次打开番茄专注 Dock 会自动恢复进行中的番茄钟（未到时的会继续倒计时，已暂停的保持暂停）。

## 专注记录格式

专注完成后，插件会在对应事项块下插入一行 Markdown，解析为番茄钟记录。你也可以在笔记中按相同格式手写记录。

### 格式一（无暂停）

```text
🍅YYYY-MM-DD HH:mm:ss~HH:mm:ss
```

- 日期与开始、结束时间；`~` 连接起止时间。
- 示例：`🍅2026-03-10 09:00:00~09:25:00`

### 格式二（带实际时长）

当专注过程中有过暂停时，会写入实际专注分钟数：

```text
🍅N,YYYY-MM-DD HH:mm:ss~HH:mm:ss
```

- `N` 为实际专注分钟数（整数），逗号可为中文或英文。
- 示例：`🍅20,2026-03-10 09:00:00~09:30:00` 表示 30 分钟内实际专注 20 分钟。

### 解析与关联规则

- 番茄钟块写在**某个事项块下方**时，会关联到该事项；写在**任务块下方**（且不在某事项下）时关联到该任务；否则关联到项目。插件据此在统计与「专注记录」列表中显示归属。

更多格式细节见 [数据格式 - 番茄钟记录](./data-format.md#番茄钟记录)。

## 通知与权限

- 专注结束时，若浏览器/思源允许通知，会弹出系统级提醒；点击通知可聚焦思源窗口。
- 首次使用番茄钟时，插件可能会请求通知权限，允许后即可收到完成提醒。

## 常见问题

### 为什么「今日番茄」为 0？

统计来自配置目录内项目文档中已解析的番茄钟记录。请确认：① 已配置目录并保存；② 项目文档中有以 `🍅` 开头的记录行，且格式正确；③ 点击 Dock 内刷新按钮重新加载数据。

### 专注记录写到哪里了？

会追加到**当前专注事项所在块**的下一行（子块）。在笔记中打开该文档，在对应事项下方即可看到新插入的 `🍅...` 行。

### 可以手动写番茄钟记录吗？

可以。在事项块下方按上述格式添加以 `🍅` 开头的行即可，插件扫描后会纳入统计与专注记录列表；记录列表点击可跳转到该块。
