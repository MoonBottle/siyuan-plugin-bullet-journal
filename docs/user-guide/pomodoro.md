# 番茄钟功能

本文档详细介绍子弹笔记插件的番茄钟功能，包括已实现的统计展示功能和即将实现的专注计时功能。

## 功能概述

番茄钟功能帮助用户追踪专注时间，提供以下能力：

1. **番茄钟记录** - 在笔记中手动添加番茄钟记录
2. **统计分析** - 查看今日/总专注时长和番茄数
3. **专注计时**（新增）- 在插件中直接开始专注倒计时

## 数据格式

### 番茄钟记录格式

在事项下方添加番茄钟行，使用 `🍅` 标记：

```markdown
- 工作事项 @2026-03-08
🍅2026-03-08 15:45:32~15:45:36 专注描述
```

格式说明：

| 元素 | 格式 | 说明 |
|------|------|------|
| 标记 | `🍅` | 番茄钟标识 |
| 日期 | `YYYY-MM-DD` | 番茄钟日期 |
| 开始时间 | `HH:mm:ss` | 专注开始时间 |
| 结束时间 | `HH:mm:ss` | 专注结束时间（可选）|
| 描述 | 任意文本 | 专注内容描述（可选）|

### 完整格式示例

```markdown
## 项目开发 #任务 @L1

- 需求分析 @2026-03-08
🍅2026-03-08 09:00:00~09:25:00 分析用户登录需求
🍅2026-03-08 10:00:00~10:25:00 整理功能清单

- 编码开发 @2026-03-08
🍅2026-03-08 14:00:00~14:25:00 实现登录接口
```

### 无结束时间格式

如果专注正在进行中或忘记记录结束时间，可以只写开始时间，默认按25分钟计算：

```markdown
🍅2026-03-08 15:45:32
```

### 列表格式支持

番茄钟行可以以列表形式书写：

```markdown
- 🍅2026-03-08 15:45:32~15:45:36 专注描述
1. 🍅2026-03-08 15:45:32~15:45:36 专注描述
```

## 块属性（新增）

为了实现专注计时功能，番茄钟块支持以下自定义属性：

```markdown
🍅2026-03-08 15:45:32
{: custom-pomodoro-status="running" custom-pomodoro-start="1741427132000" custom-pomodoro-duration="25" custom-pomodoro-item-id="item-xxx" custom-pomodoro-item-content="工作事项"}
```

### 属性说明

| 属性名 | 类型 | 说明 |
|--------|------|------|
| `custom-pomodoro-status` | string | 专注状态：`running`（进行中）、`completed`（已完成）|
| `custom-pomodoro-start` | number | 开始时间戳（毫秒）|
| `custom-pomodoro-duration` | number | 设定专注时长（分钟）|
| `custom-pomodoro-item-id` | string | 关联事项 ID |
| `custom-pomodoro-item-content` | string | 关联事项内容 |

### Kramdown 语法

思源笔记使用 Kramdown 格式，块属性语法为：

```markdown
{: 属性名1="值1" 属性名2="值2"}
```

属性写在块内容之后，单独一行。

## 功能模块

### 1. 番茄钟统计面板（已实现）

位置：侧边栏番茄钟 Dock

功能：
- **今日番茄数** - 今日完成的番茄钟数量
- **今日专注时长** - 今日专注总时长（格式：25m 或 1h 30m）
- **总番茄数** - 所有时间累计的番茄数
- **总专注时长** - 所有时间累计的专注时长

#### 统计卡片布局

```
┌─────────────────┬─────────────────┐
│   今日番茄      │   今日专注时长  │
│      6          │     1h 42m      │
├─────────────────┼─────────────────┤
│   总番茄        │   总专注时长    │
│      6          │     1h 42m      │
└─────────────────┴─────────────────┘
```

### 2. 专注记录列表（已实现）

按日期分组展示所有番茄钟记录：

```
专注记录 [开始专注]  ← 新增按钮

3月8日
🍅 09:00 - 09:25    需求分析        25m
🍅 10:00 - 10:25    需求分析        25m
🍅 14:00 - 14:25    编码开发        25m

3月7日
🍅 15:00 - 15:25    文档整理        25m
```

每条记录显示：
- 番茄图标 🍅
- 时间范围（HH:mm - HH:mm）
- 关联事项/任务名称
- 专注描述（如有）
- 时长（分钟）

点击记录可跳转到思源笔记对应位置。

### 3. 开始专注弹框（新增）

点击"开始专注"按钮打开弹框：

```
┌─────────────────────────────────────────┐
│              开始专注                    │
├─────────────────────┬───────────────────┤
│   选择待办事项       │   设置专注时长     │
│                     │                   │
│ ⚠️ 过期事项          │  [15分] [25分]    │
│ ○ 需求分析 @03-06   │  [45分] [60分]    │
│ ○ 代码评审 @03-07   │                   │
│                     │  自定义：____ 分钟 │
│ 📅 今天事项          │                   │
│ ● 接口开发 @03-08   │                   │
│ ○ 文档编写 @03-08   │    [ 开始专注 ]   │
│                     │                   │
└─────────────────────┴───────────────────┘
```

#### 左侧：待办事项列表

- **过期事项** - 日期 < 今天 且状态为待办的事项
- **今天事项** - 日期 = 今天 且状态为待办的事项
- 单选，必须选择一个事项才能开始
- 选中项高亮显示

#### 右侧：专注时长设置

- 快捷按钮：15分钟、25分钟（默认）、45分钟、60分钟
- 自定义输入：1-180 分钟

### 4. 专注中状态（新增）

开始专注后，Dock 切换为专注状态展示：

```
┌─────────────────────────────────┐
│  ⏱️ 专注中                       │
│                                 │
│      ╭─────────────╮            │
│     ╱   15:32     ╱             │
│    ╱  接口开发    ╱              │
│   ╱─────────────╱               │
│                                 │
│   [ 结束专注 ]  [ 取消 ]        │
└─────────────────────────────────┘
```

显示内容：
- 倒计时（MM:SS）
- 当前专注事项名称
- 结束专注按钮（提前结束，删除番茄钟行）
- 取消按钮（取消专注，删除番茄钟行）

倒计时结束后：
- 播放提示音
- 显示完成通知
- 自动更新块内容（添加结束时间）
- 更新块属性 `custom-pomodoro-status` 为 `completed`
- Dock 回归正常展示

## 技术实现

### 数据流

```
用户点击"开始专注"
    │
    ▼
打开 PomodoroTimerDialog
    │
    ▼
选择事项 + 设置时长
    │
    ▼
调用 pomodoroStore.startPomodoro()
    │
    ├──► 调用思源 appendBlock 创建番茄钟块
    ├──► 调用思源 setBlockAttrs 设置属性
    └──► 启动倒计时定时器
    │
    ▼
Dock 展示 PomodoroActiveTimer
    │
    ▼
倒计时结束 / 用户结束 / 用户取消
    │
    ├──► 完成：updateBlock + setBlockAttrs
    └──► 取消/提前结束：deleteBlock
```

### 思源 API 调用

| 操作 | API | 用途 |
|------|-----|------|
| 创建番茄钟块 | `appendBlock` | 在事项下添加番茄钟行 |
| 设置块属性 | `setBlockAttrs` | 保存专注状态、开始时间、时长等 |
| 获取块属性 | `getBlockAttrs` | 恢复专注状态时读取 |
| 更新块内容 | `updateBlock` | 完成时添加结束时间 |
| 删除块 | `deleteBlock` | 提前结束或取消时删除 |

### 状态恢复

插件加载时：

1. 遍历所有番茄钟记录
2. 查找 `custom-pomodoro-status="running"` 的块
3. 如果有，计算剩余时间 = 设定时长 - (当前时间 - 开始时间)
4. 如果剩余时间 > 0，恢复倒计时
5. 如果剩余时间 <= 0，自动标记为完成

## 使用流程

### 手动记录番茄钟（已有功能）

1. 在事项下方手动添加番茄钟行
2. 格式：`🍅YYYY-MM-DD HH:mm:ss~HH:mm:ss 描述`
3. 查看 Dock 统计和记录列表

### 使用专注计时（新增功能）

1. 点击 Dock 中的"开始专注"按钮
2. 在弹框中选择一个待办事项
3. 设置专注时长（或保持默认25分钟）
4. 点击"开始专注"
5. 专注过程中 Dock 显示倒计时
6. 专注结束后自动记录到笔记

## 注意事项

1. **状态持久化** - 专注状态通过思源块属性保存，重启插件后可恢复
2. **跨设备同步** - 块属性随笔记同步，可在不同设备上恢复专注状态
3. **提前结束** - 提前结束会删除番茄钟行，不会保留记录
4. **专注完成** - 正常完成的番茄钟会保留在笔记中，包含完整的开始和结束时间
