# 设计思想

本文档介绍子弹笔记插件的设计理念，帮助您理解插件的核心思想和使用方式。

## 与传统待办工具的区别

本插件的设计理念与传统待办工具（如滴答清单、Todoist）有本质不同，更接近**子弹笔记（Bullet Journal）**的思路：

| 传统待办工具 | 子弹笔记思路（本插件） |
|-------------|----------------------|
| 强调提醒和定时 | 强调记录和回顾 |
| 支持重复任务 | **不支持重复任务** |
| 任务可以跨越多天 | **事项不跨天** |
| 提前规划大量未来任务 | **聚焦今日及未来的待办** |
| 提醒驱动 | 记录驱动 |

## 核心定位

### 记录 > 提醒

不是为了推送提醒，而是记录你今日及未来要完成或已完成的事。

传统待办工具的核心是「提醒」，你需要设置时间，系统会在指定时间推送通知。这种方式适合有明确截止时间的任务，但不适合日常工作记录。

子弹笔记的核心是「记录」，你记录下要做的事和已完成的事，通过回顾来管理任务。这种方式更适合知识工作者，因为很多工作难以预估时间。

### 分散书写，集中查看

把分散在各处的想法统一记录，通过日历/甘特图/项目列表集中回顾。

在思源笔记中，你可以在任何文档中记录任务，插件会自动扫描并汇总到各个视图中。这样你可以：

- 在项目文档中记录项目相关任务
- 在会议笔记中记录行动项
- 在日记中记录日常待办

所有任务都会自动出现在日历、甘特图和项目列表中。

### 视图即回顾

日历看今日事项，甘特图看进度，项目看归类，视图是梳理工具而非提醒工具。

不同的视图提供不同的视角：

- **日历**：今天要做什么？本周有哪些安排？
- **甘特图**：项目进度如何？哪些任务有依赖？
- **项目列表**：这个项目有哪些任务？都完成了吗？

通过切换视图，你可以从不同角度审视自己的工作。

### 事项不跨天

今日事今日毕。

每个事项只属于一天。如果一个任务需要多天完成，就创建多个事项：

```markdown
开发登录模块 #任务 @L1
需求评审 @2026-02-20
编码开发 @2026-02-21
测试验收 @2026-02-22
```

这种方式的好处是：

- 每天都有明确的待办
- 容易追踪每日完成情况
- 避免「长期任务」被遗忘

## 无侵入式设计

本插件采用**无侵入式设计**，不修改用户的笔记内容，而是通过解析现有的 Markdown 格式来提取任务信息。这意味着：

- 您的笔记仍然是纯 Markdown 文件，可以在其他编辑器中正常阅读
- 不需要学习新的语法，仅使用标准的 Markdown 标记
- 数据完全存储在思源笔记中，无需额外的数据库

### 为什么选择无侵入式？

1. **数据主权**：你的数据完全属于你，不依赖任何专有格式
2. **工具自由**：即使不使用本插件，笔记仍然可读可用
3. **迁移友好**：随时可以导出到其他工具

## 块级双向链接

利用思源笔记的 Kramdown 格式，插件可以获取每个段落的唯一 `blockId`，实现：

- 点击日历/甘特图中的任务 → 跳转到对应的文档位置
- 在文档中修改任务内容 → 自动同步到各个视图
- 每个任务都有唯一的持久化链接

### 为什么需要双向链接？

传统待办工具的问题是：待办列表和笔记是分离的。你在待办列表中看到一条任务，但想查看详情时需要去别的地方找。

子弹笔记插件通过双向链接解决了这个问题：

- 在日历中看到任务 → 点击直接跳转到笔记
- 在笔记中修改 → 自动同步到所有视图

任务和上下文始终在一起。

## 不支持的功能

基于设计理念，以下功能**不会支持**：

| 功能 | 原因 |
|------|------|
| 重复任务 | 记录驱动，每个事项都是独特的 |
| 提醒推送 | 视图即回顾，不需要提醒 |
| 任务跨天 | 事项不跨天，今日事今日毕 |
| 子任务嵌套 | 保持简单，最多三级任务 |

如果你需要这些功能，建议使用传统待办工具，或将本插件与传统工具配合使用。
