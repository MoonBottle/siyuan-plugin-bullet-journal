# 配置说明

本文档介绍任务助手插件的各项配置选项。

## 设置界面

打开设置：思源设置 → 插件 → 任务助手 → 设置

设置界面分为左右两栏：
- **左侧菜单**：快速导航到各配置章节
- **右侧内容**：具体配置选项

支持搜索功能，可快速定位配置项。

## 目录配置

目录配置决定插件扫描哪些文档来提取任务信息。

> 🆕 **v0.12.2 新增**：扫描范围设置（全局扫描/仅配置目录），解决配置目录后任务消失的问题。

### 扫描范围（v0.12.2 新增）

插件支持两种扫描模式，可在「目录配置」顶部切换：

| 模式 | 图标 | 说明 | 适用场景 |
|------|------|------|----------|
| **扫描整个工作空间** | 🌐 | 扫描所有包含任务标记的文档，目录配置仅用于分组归类 | 日常使用，推荐 |
| **仅扫描配置目录** | 📁 | 只扫描下方配置的目录中的文档 | 大型工作空间，性能优化 |

#### 扫描范围与目录配置的关系

- **全局扫描模式**（默认）：扫描所有文档，配置的目录用于**分组归类**
  - 所有任务都可见
  - 匹配目录配置的项目会显示在对应分组
  - 未匹配的项目显示为「未分组」

- **仅配置目录模式**：只扫描配置的目录，与 v0.12.2 之前的行为一致
  - 只显示配置目录中的任务
  - 其他文档的任务不会被扫描

#### 升级用户注意事项

从旧版本升级到 v0.12.2 后：
- 默认切换为「全局扫描模式」，之前消失的任务会重新出现
- 如需恢复原行为，请手动切换到「仅扫描配置目录」模式

### 添加扫描目录

1. 确保扫描模式为「仅扫描配置目录」（全局扫描模式下目录配置仅用于分组）
2. 在「目录配置」区域点击「添加目录」
3. 输入目录路径（如 `工作安排/2026/项目`）
4. 选择所属分组（可选）
5. 保存设置

![目录配置截图](./images/configuration-directory.png)

### 目录配置项

| 配置项 | 说明 |
|--------|------|
| 路径 | 文档所在目录的相对路径 |
| 分组 | 该目录下项目的默认分组 |
| 启用 | 是否启用该目录的扫描/分组 |

### 路径格式

- 使用相对路径，相对于思源笔记本根目录
- 支持多级目录，用 `/` 分隔
- 路径不需要以 `/` 开头或结尾

### 示例

| 路径 | 说明 |
|------|------|
| `工作安排/2026/项目` | 扫描指定目录下的文档 |
| `工作安排` | 扫描工作安排及其所有子目录 |
| `日记` | 扫描日记目录 |

## 分组管理

分组功能帮助您组织和筛选项目。

### 默认分组

可设置一个默认分组，新目录自动归属到该分组。

### 创建分组

1. 打开插件设置
2. 在「分组管理」区域点击「添加分组」
3. 输入分组名称（如「前端项目」「后端项目」）
4. 为目录分配分组

### 分组用途

- 在项目列表中按分组筛选
- 在甘特图中按分组查看
- 组织和归类不同类型的项目

## 番茄钟

番茄钟专注计时的相关设置。

### 界面设置

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| 悬浮番茄按钮 | 专注时在右下角显示悬浮按钮 | 关闭 |
| 底栏倒计时 | 专注时在底栏显示倒计时面板（含继续/暂停控制） | 关闭 |
| 底栏进度条 | 专注时在页面底部显示进度条 | 关闭 |

### 记录存储方式

番茄钟记录的存储位置：

| 模式 | 说明 |
|------|------|
| 事项块下（子块） | 在事项下方创建子块存储记录 |
| 事项块自定义属性 | 存储在块的自定义属性中 |

### 自动结束专注

- **事项完成时自动结束番茄钟**：在待办面板标记事项完成时，自动结束关联的番茄钟

### 最小专注时间

专注时长低于此值时将提醒用户（分钟），范围 1-60 分钟，默认 5 分钟。

### 自动延迟设置

倒计时结束时自动延长专注时间：

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| 自动延迟 | 倒计时结束且未操作弹窗时，自动延长倒计时继续专注 | 关闭 |
| 等待时间（秒） | 弹窗弹出后等待多少秒触发自动延迟 | 30 秒 |
| 延长分钟数 | 每次自动延迟增加的倒计时分钟数 | 5 分钟 |
| 最大延迟次数 | 达到最大次数后弹窗将保持打开 | 3 次 |

## 午休时间

午休时间用于计算工作时长时扣除午休时段，影响甘特图中的任务时长显示。

### 配置方式

1. 打开插件设置
2. 找到「午休时间」设置
3. 设置开始时间和结束时间（默认 12:00-13:00）

### 计算规则

当事项时间范围跨越午休时段时，会自动扣除午休时间：

```
事项：09:00 ~ 14:00
午休：12:00 ~ 13:00
实际工作时长：4 小时（而非 5 小时）
```

## 斜杠命令

自定义斜杠命令快捷方式。

### 内置命令

插件提供以下内置斜杠命令：

| 命令 | 功能 |
|------|------|
| `/today`, `/jt` | 标记为今日事项 |
| `/tomorrow`, `/mt` | 标记为明日事项 |
| `/date`, `/rq` | 选择日期 |
| `/done`, `/wc` | 标记为已完成 |
| `/abandon`, `/fq` | 标记为已放弃 |
| `/calendar`, `/rl` | 打开日历 |
| `/calendarday`, `/rlt` | 打开日历-天视图 |
| `/calendarweek`, `/rlz` | 打开日历-周视图 |
| `/calendarmonth`, `/rly` | 打开日历-月视图 |
| `/calendarlist`, `/rll` | 打开日历-列表视图 |
| `/gantt`, `/gt` | 打开甘特图 |
| `/focus`, `/zz` | 开始专注 |
| `/todo`, `/db` | 打开待办 Dock |
| `/projectdir`, `/ml` | 设置为项目目录 |
| `/task`, `/rw` | 标记为任务 |
| `/detail`, `/xq` | 查看详情 |

### 自定义命令

可添加自定义斜杠命令：

1. 点击「添加命令」
2. 输入命令名称（如「快速标记今天」）
3. 输入触发命令（如 `/my`, `/我的`）
4. 选择执行动作
5. 保存

## AI 服务配置

配置大语言模型服务，与任务助手进行智能对话。

### 启用 AI 对话

开启后可在 AI 对话面板与 AI 助手交互。

### 添加供应商

1. 点击「添加供应商」
2. 输入配置名称（如：Kimi-个人）
3. 选择供应商或选择「自定义」
4. 输入 API 地址
5. 输入 API Key
6. 添加模型并勾选启用
7. 保存

### 模型管理

- 可添加多个模型
- 勾选要启用的模型
- 设置默认模型

### 对话设置

- **显示工具调用详情**：在对话中展示 AI 工具调用的详细信息和执行结果

## MCP 配置

用于让支持 MCP 的 AI 助手（如 Trae、Cursor）访问任务与番茄钟数据。

### 配置步骤

1. 打开思源设置 → 插件 → 任务助手，在「MCP 配置」区域点击「复制 MCP 配置」
2. 在 AI 助手的 MCP 设置中添加该服务器，将配置中的 `SIYUAN_TOKEN` 替换为思源的 API Token（思源 → 设置 → 关于 → API Token）
3. 多工作空间时，如需指定端口，可修改 `SIYUAN_API_URL`（默认 `http://127.0.0.1:6806`）

详细步骤与工具说明见 [MCP AI 助手](./mcp.md)。

## 常见问题

### Q: 为什么看不到任务数据？

请检查：
1. 扫描范围设置是否正确（全局扫描模式下应能看到所有任务）
2. 目录路径是否与实际文件夹结构匹配（仅配置目录模式）
3. 项目文档是否包含 `#任务` 标记
4. 工作事项是否有有效的日期格式（`@YYYY-MM-DD`）

### Q: 如何扫描多个目录？

在目录配置中添加多个目录路径即可。每个目录可以分配到不同的分组。

### Q: 目录修改后需要重启吗？

不需要。保存设置后插件会自动重新扫描。

### Q: 如何查看扫描结果？

在项目列表视图中可以看到所有扫描到的项目和任务。

### Q: 分组有什么作用？

分组用于组织和筛选项目：
- 在项目列表中按分组筛选
- 在甘特图中按分组查看
- 配合扫描范围实现项目的分类管理
