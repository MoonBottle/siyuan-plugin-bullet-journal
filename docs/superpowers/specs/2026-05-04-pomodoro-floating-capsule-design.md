# Pomodoro Floating Capsule Design

## 背景

当前番茄钟悬浮窗是一个右下角圆形按钮，只承载两类能力：

- 显示当前计时的 `MM:SS`
- 点击后打开或切换 Pomodoro Dock

它已经满足“计时不丢”和“快速回到番茄面板”这两个基础目标，但也有明显短板：

- 视觉上更像一个普通悬浮按钮，而不是“当前专注状态”
- 缺少事项上下文，用户无法一眼知道自己在专注什么
- 缺少就地控制，暂停、继续、结束、跳过休息仍需依赖底栏或 Dock
- 在桌面端信息密度偏低，悬浮层的使用价值没有充分释放

## 目标

将当前番茄钟悬浮窗升级为 **双态胶囊式悬浮条**，同时提升：

- 视觉完成度：从圆形按钮升级为更克制、更像桌面工具的状态胶囊
- 信息完整度：在展开态中补齐事项名、状态、副信息
- 操作便利性：允许用户直接在悬浮胶囊中做核心控制

## 非目标

- 不修改 `pomodoroStore` 的核心计时、恢复、完成、休息逻辑
- 不重做 Pomodoro Dock 主界面
- 不在本次设计中增加新的番茄业务状态或新的计时模式
- 不为移动端新增复杂展开交互
- 不替换底栏倒计时与底栏进度条的现有能力

## 决策

采用 **双态胶囊 + 桌面端悬停展开**。

悬浮番茄钟分为两种形态：

1. `默认态（迷你胶囊）`
2. `展开态（完整胶囊）`

默认态保持低干扰，只显示图标与时间；展开态提供上下文信息与快捷控制。桌面端通过 `hover` 展开，移动端继续保持“点击进入 Pomodoro 视图”的现有行为。

## 交互设计

### 默认态

- 形态为横向迷你胶囊，不再使用纯圆
- 内容只保留：
  - 左侧图标
  - 右侧时间
- 点击主体：
  - 打开或切换 Pomodoro Dock
- 支持拖拽
- 拖拽中不触发展开和点击打开

### 展开态

桌面端鼠标悬停时，胶囊从默认态向左展开，展示完整内容。

展开态分为三段：

- 左侧：图标 + 状态标签
- 中间：事项名 + 副信息
- 右侧：操作按钮

点击规则：

- 点击胶囊主体空白区域：打开或切换 Pomodoro Dock
- 点击右侧操作按钮：只触发对应动作，不冒泡为打开 Dock

### 操作分层

主操作：

- `暂停 / 继续`

次操作：

- `结束专注`
- `跳过休息`

显示规则：

- 专注中：显示 `暂停 / 继续` 和 `结束专注`
- 休息中：显示 `跳过休息`
- 无进行中的计时：不显示悬浮胶囊

## 信息结构

### 主信息

主信息始终为当前计时核心读数：

- 倒计时专注：显示剩余时间
- 正计时专注：显示累计时间
- 休息中：显示剩余休息时间

### 副信息

副信息根据计时模式切换：

- 倒计时专注：`已专注 X 分钟 / 目标 Y 分钟`
- 正计时专注：`已专注 X 分钟`
- 休息中：`休息剩余 X 分钟`

副信息不写死为单一“已专注分钟数”，必须根据当前计时类型动态映射。

### 事项名

- 展开态显示当前事项名
- 单行显示，超出宽度后截断
- 若无法从当前 active pomodoro 关联到完整事项，回退为已有 `itemContent`
- 若仍为空，则使用兜底文案，避免出现空白内容区

### 状态标签

状态标签附着在图标旁，不单独占据一整行，用于快速区分：

- `专注中`
- `已暂停`
- `休息中`

`正计时` 与 `倒计时` 不使用独立颜色做模式区分，而通过副信息文案表达，避免状态颜色语义过载。

## 视觉设计

### 整体风格

视觉上采用偏工具型而非装饰型的设计，尽量贴近 SiYuan 桌面环境：

- 整体为完整胶囊
- 阴影轻量，不做强悬浮广告感
- 不使用整块高饱和主色填满背景
- 使用主题表面色作为主体背景，主色仅承担强调作用

### 尺寸建议

- 默认态高度：约 `40-44px`
- 默认态宽度：约 `92-108px`
- 展开态宽度：约 `220-280px`
- 圆角半径：接近高度一半，形成真正胶囊感

### 内容层级

- 时间：最大层级，优先保证可读性
- 事项名：第二层级，单行截断
- 副信息：较弱层级，小字号显示
- 操作：图标按钮优先，不使用长文本按钮，避免撑宽

### 状态颜色

- `专注中`：使用主题主色或偏红橙的专注强调色
- `已暂停`：使用中性偏黄的提示色
- `休息中`：使用偏绿或青色

颜色用于传达“当前状态”，而不是传达“计时模式”。

### 进度表达

- 展开态底部增加一条细进度线
- 倒计时专注：按目标时长显示完成进度
- 正计时专注：仍显示进度，但参考值沿用当前设计中的固定基准
- 休息中：使用休息剩余时间的反向进度

进度线只作为辅助视觉反馈，不取代主时间读数。

## 平台差异

### 桌面端

- 支持悬停展开
- 支持拖拽
- 支持胶囊内快捷操作

### 移动端

- 不引入 hover 展开
- 保持当前行为：点击进入 Pomodoro 相关视图
- 若需要胶囊视觉统一，可仅复用默认态外观，不复用展开交互

## 实现边界

本次改动应尽量限制在悬浮胶囊 UI 层，不改核心业务流。

建议主要落在以下已有结构：

- `src/index.ts`
  - `createFloatingTomatoButton()`
  - `makeDraggable()`
  - `showFloatingTomatoButton()`
  - `updateTimerDisplaysFromStore()`
- `src/index.scss`
  - `.floating-tomato-btn` 及相关子节点样式

## 实现拆分建议

### 1. DOM 结构升级

`createFloatingTomatoButton()` 不再只创建“图标 + 时间”，而是一次性创建默认态与展开态共用结构，避免 hover 时反复重建节点。

建议包含：

- 图标区
- 时间区
- 状态标签区
- 事项名区
- 副信息区
- 主操作按钮区
- 次操作按钮区
- 底部进度线

### 2. 状态映射集中化

`updateTimerDisplaysFromStore()` 应从“只更新时间”扩展为统一产出一份 UI 状态。

建议内部先整理出：

```ts
type FloatingPomodoroViewState = {
  phase: 'focus' | 'break'
  timerMode?: 'countdown' | 'stopwatch'
  isPaused?: boolean
  primaryText: string
  secondaryText: string
  itemTitle: string
  progress: number
  canPauseOrResume: boolean
  canComplete: boolean
  canSkipBreak: boolean
}
```

再将这份状态映射到 DOM，避免零散地在多个分支里直接操作节点。

### 3. 交互控制

拖拽与展开需要明确互斥：

- 开始拖拽后加 `dragging` class
- `dragging` 状态下禁止展开
- 鼠标抬起后短延迟移除 `dragging`，避免拖拽尾声误触 hover

同时需要确保：

- 操作按钮点击阻止事件冒泡
- 主体点击与按钮点击不会串行为

### 4. 样式层实现

样式主要集中在 `.floating-tomato-btn` 区域：

- 默认态与展开态尺寸切换
- hover 动画
- 状态标签样式
- 图标按钮样式
- 文本截断
- 进度线样式

动画以轻量过渡为主，不使用复杂弹跳或位移动画。

## 行为规则

### 显示与隐藏

- 专注中：显示悬浮胶囊
- 休息中：显示悬浮胶囊
- 倒计时已结束且进入完成态：按现有逻辑隐藏
- 无活动番茄且无休息：隐藏

### 恢复逻辑

恢复已有番茄时：

- 悬浮胶囊应立即根据 store 状态渲染
- 不依赖下一次 tick 才填充完整信息

### 事项回退

事项标题读取顺序建议为：

1. 当前 active pomodoro 关联的完整事项内容
2. `activePomodoro.itemContent`
3. 兜底文案

### 暂停态

暂停时：

- 状态标签切换为 `已暂停`
- 主操作切换为 `继续`
- 时间保持当前值，不继续流动

### 休息态

休息时：

- 图标切换为休息相关图标
- 状态标签切换为 `休息中`
- 主体时间显示休息剩余时间
- 右侧不显示“结束专注”，只显示“跳过休息”

## 风险与约束

### 拖拽与 Hover 冲突

双态胶囊最大风险是拖拽与悬停交互冲突。需要通过 `dragging` 状态和延迟释放来规避误展开与误点击。

### 信息拥挤

展开态内容较多，必须控制：

- 事项名单行截断
- 按钮图标化
- 胶囊总宽度上限

否则容易从“信息更完整”退化成“信息堆叠”。

### 业务逻辑边界

本次不应把新的计时判断逻辑散落在悬浮胶囊实现中。所有新增 UI 状态都应从现有 store 状态推导，而不是引入新的独立状态源。

## 测试要求

### 状态显示

- 倒计时专注时，显示剩余时间与 `已专注 / 目标`
- 正计时专注时，显示累计时间与 `已专注`
- 暂停时，状态、按钮、时间表现正确
- 休息中时，图标、按钮、副信息正确切换

### 交互行为

- 点击主体可打开或切换 Pomodoro Dock
- 点击 `暂停 / 继续` 只触发对应动作
- 点击 `结束专注 / 跳过休息` 只触发对应动作
- 拖拽后不会误打开 Dock
- 拖拽过程中不会触发展开

### 边界情况

- 无事项名时有稳定兜底显示
- 恢复中的番茄能立即显示完整信息
- 计时结束瞬间不会残留脏状态
- 切换到休息态时按钮和文案不会沿用专注态旧状态

## 验收标准

1. 桌面端悬浮番茄钟从圆形按钮升级为胶囊形态
2. 默认态只显示图标与时间，保持低干扰
3. 悬停展开后可看到事项名、状态、副信息和快捷操作
4. 倒计时、正计时、休息态的副信息按模式正确切换
5. 用户可直接在胶囊中完成 `暂停 / 继续` 与 `结束专注 / 跳过休息`
6. 拖拽、点击主体、点击操作按钮三类交互互不冲突
7. 移动端不引入新的复杂展开交互，保持现有使用路径
