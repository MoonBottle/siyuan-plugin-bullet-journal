# 专注页计划环节设计

日期：2026-05-14
状态：设计已确认，可进入实现计划
范围：在现有专注复盘页中补充“计划”环节，并重新评估页面命名

## 1. 背景

当前专注相关能力已经具备两段闭环：

1. 执行：事项可直接开始专注，番茄钟流程完整
2. 复盘：按日查看有预计事项的实际、偏差、状态与记录

仍然缺少的是“计划”入口。

虽然系统已经通过 `/focusplan` 和 `/yj` 提供了“为当前事项设置预计”的能力，但这个入口依赖用户先在文档中定位到某一条事项。当前专注页里还无法直接完成以下动作：

- 从候选事项中挑选一个事项
- 为该事项新增预计
- 对当前选中事项修改预计

这会导致用户在“计划 -> 执行 -> 复盘”的链路中，仍然需要频繁跳回文档编辑器才能开始计划。

本设计的目标是在不改变“文档标记即真源”和不新建一套预计模型的前提下，把 `/yj` 背后的能力接入当前专注页，形成页内闭环。

## 2. 目标

本次设计的目标是：

1. 在当前专注页中补充“为事项设置预计”的计划入口
2. 新增预计时允许用户先选择事项，再复用现有预计编辑器
3. 编辑预计时允许用户从当前事项操作区直接进入
4. 保存后即时刷新当前日期列表、摘要和详情
5. 让页面语义从“只做复盘”升级为“计划 + 执行 + 复盘”的完整工作面

## 3. 非目标

本次明确不做：

- 创建新事项
- 改造预计标记格式
- 新做一套预计编辑弹窗
- 将“无预计事项”直接混入当前左侧复盘列表
- 扩展候选范围到全量事项搜索
- 单独持久化“今日计划清单”

## 4. 产品决策

### 4.1 计划入口仍附着在事项上

“计划”在本设计里指的是“为事项设置或修改预计”，不是单独创建一个计划对象。

原因：

- 当前预计模型本来就是事项级
- `/yj` 已经验证了这个交互方向
- 避免额外引入“计划项”和“事项”两套对象

### 4.2 新增预计与修改预计分两条路径

页面内保留两条入口：

1. 新增预计：从左侧列表区的工具入口进入，先选事项
2. 修改预计：从右侧当前事项操作区进入，直接编辑当前事项

原因：

- 新增预计时，用户还没有绑定到某个事项
- 修改预计时，用户已经明确聚焦于当前事项

### 4.3 候选事项范围受限

“新增预计”的候选事项只包含：

1. 过期事项
2. 当前选中日期对应的事项

不包含：

- 全量事项
- 任意日期历史事项
- 仅通过搜索召回的脱离日期上下文事项

原因：

- 保持专注页的“按日计划与复盘”语义
- 避免页面退化成第二个 Todo 面板
- 减少候选过多带来的选择负担

### 4.4 页面仍以复盘为主，不变成 Todo 页

左侧现有列表仍然表示“当前日期有预计的事项”，不改成“所有事项列表”。

计划能力以工具入口和空状态入口承接，而不是改写主列表语义。

## 5. 交互设计

### 5.1 左侧新增预计入口

在左侧状态 pills 下方、事项列表上方增加一个轻量入口：

- 文案：`添加预计`
- 形式：列表工具条中的文本按钮或轻按钮
- 视觉优先级低于日期、统计和筛选，不抢复盘主视觉

该入口用于“当前页面里还没有锁定某个事项”的场景。

### 5.2 新增预计流程

流程如下：

1. 用户点击 `添加预计`
2. 打开一个轻量选择面板
3. 面板展示两组候选事项：
   - `过期事项`
   - `所选日期事项`
4. 用户点击某个候选事项
5. 直接打开现有 `/yj` 同款预计编辑器
6. 用户保存后，返回当前专注页并触发数据刷新

不增加中间确认步骤，不跳新页面。

### 5.3 候选事项选择面板

选择面板的目标是“快速锁定事项”，不是展示完整详情。

每条候选项只展示：

- 事项内容
- 所属任务 / 项目（弱化样式）
- 日期 / 时间
- 当前预计（若已存在）

规则：

- 没有数据的分组不显示
- 整行可点击
- 已有预计的事项允许继续进入，用于“从新增入口补做修改”

### 5.4 右侧修改预计入口

在当前 `ItemActionBar` 中新增一个与 Todo 风格一致的图标按钮。

位置建议：

1. 完成
2. 开始专注
3. 设置 / 修改预计
4. 迁移
5. 放弃
6. 打开文档
7. 日历

tooltip：

- 无预计：`设置预计`
- 有预计：`修改预计`

### 5.5 图标选择

根据 [icons.md](C:/dev/projects/open-source/siyuan-plugin-bullet-journal/docs/API/icons.md)，本次优先建议使用：

- `#iconAttr`

理由：

- 语义更接近“编辑事项元信息”
- 能与 `时钟 / 迁移 / 放弃 / 日历` 这些动作图标保持区分
- 比 `#iconEdit` 更少“通用编辑”歧义

备选：

- `#iconEdit`：语义直接，但较泛
- `#iconFocus`：更接近执行，不适合代表“设置预计”

### 5.6 空状态承接

当当前选中日期还没有任何预计事项时：

- 左侧空态标题：`还没有预计事项`
- 左侧空态描述：`为过期事项或当前日期事项设置预计后，这里会显示专注复盘。`
- 空态中提供一个入口：`为事项设置预计`

这保持了左侧区域仍以“有预计事项列表”为主语义，同时给出进入计划动作的路径。

## 6. 数据与状态规则

### 6.1 候选事项过滤规则

候选事项必须满足：

- 有 `blockId`
- 不是已完成事项
- 不是已放弃事项
- 属于以下任一集合：
  - 过期事项
  - 当前选中日期的事项

### 6.2 候选事项排序规则

排序建议如下：

1. 先显示过期事项，按日期从早到晚
2. 再显示当前选中日期事项
3. 同一天内，先按开始时间排序；无时间的事项排在后面

### 6.3 保存后的刷新行为

保存预计后：

- 重新获取当前日期的复盘摘要
- 重新计算左侧预计事项列表
- 重新渲染右侧详情区

具体行为：

- 新增预计：
  - 该事项立即进入左侧列表
  - 自动选中该事项
- 修改预计：
  - 保持当前选中项不变
  - 即时更新 `实际 / 预计` 和偏差展示
- 清除预计：
  - 该事项从左侧预计列表移除
  - 若当天已无预计事项，回到空态

## 7. 复用边界

本次不新建预计编辑能力，而是完全复用当前 `/yj` 背后的实现。

复用内容包括：

- 预计编辑弹窗
- 表单模式切换
- 番茄数 / 专注时长二选一规则
- 文档标记写回规则
- 清除预计逻辑

本次新增的只是两层页面适配壳：

1. 从复盘页打开“事项选择 -> 预计编辑”的流程
2. 从事项操作区打开“当前事项预计编辑”的流程

## 8. 异常处理

### 8.1 候选事项状态变化

如果用户打开候选面板后，某事项已经变成：

- 已完成
- 已放弃
- 块不存在

则：

- 不打开预计编辑器
- toast 提示：`当前事项状态已变化，请刷新后重试`

### 8.2 写入失败

如果预计写回失败：

- 当前页面不跳转
- 不做乐观更新
- toast 提示失败
- 等用户重试或手动刷新

### 8.3 解析异常

预计解析异常不能导致事项在复盘页失效。

保存动作仍按现有 `/yj` 规则收敛为一个合法预计标记。

## 9. 页面命名评估

### 9.1 现有命名的问题

当前页面名为“专注复盘”。

在只包含“摘要 + 偏差 + 记录”时，这个名字是准确的。
但在本次设计落地后，该页面将同时承载：

- 计划：设置 / 修改预计
- 执行：开始专注
- 复盘：查看实际、偏差、记录

继续使用“专注复盘”，会低估页面能力，也会让“左侧添加预计”和“右侧开始专注”看起来像是插入到错误页面里的动作。

### 9.2 命名备选

备选 1：`专注工作台`
优点：

- 最匹配“计划 + 执行 + 复盘”的综合工作面
- 容纳后续继续扩展而不再改名
- 与当前页面左右分区、工具条、详情区的形态一致

缺点：

- “工作台”语义更宽，需要在页面内部继续用模块标题强调复盘区

备选 2：`专注计划与复盘`
优点：

- 语义最直接
- 用户一眼能看懂页内会发生什么

缺点：

- 名称偏长
- 页面里已有“开始专注”，但标题未显式包含执行

备选 3：保留 `专注复盘`
优点：

- 不影响现有认知

缺点：

- 与新增计划、执行能力不再完全匹配

### 9.3 推荐命名

推荐将页面名改为：`专注工作台`

理由：

1. 它能覆盖计划、执行、复盘三段闭环
2. 后续如果增加更多专注相关工具，不需要再次更名
3. 具体的“复盘”语义可以留给左侧摘要区或右侧记录区的小标题来表达

如果希望更强调专注计划，也可以把页面副标题或局部标题保留为：

- `专注复盘`
- `今日复盘`
- `专注记录`

## 10. 测试重点

本次实现应覆盖以下验证点：

1. 左侧 `添加预计` 入口在有数据和空状态下都可触发
2. 候选事项只包含过期事项和当前日期事项
3. 候选事项不会包含已完成 / 已放弃事项
4. 点击候选事项后会打开现有预计编辑器
5. 右侧操作区会根据当前事项是否已有预计显示 `设置预计 / 修改预计`
6. 保存预计后左侧列表、摘要、右侧详情能即时更新
7. 清除预计后事项会从复盘列表移除
8. 写入失败和状态变化时有稳定提示，不产生脏 UI

## 11. 结论

本次设计不引入新的计划模型，而是把现有事项级预计能力接入当前专注页，补齐“计划”环节。

交互上形成两条入口：

- 左侧：为候选事项新增预计
- 右侧：对当前事项修改预计

这样页面可形成：

- 计划
- 执行
- 复盘

的完整闭环，因此页面名推荐从 `专注复盘` 升级为 `专注工作台`。
