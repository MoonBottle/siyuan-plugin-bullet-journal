# README 优化设计方案

## 背景

当前 `README.zh-CN.md` 内容停留在 v0.12.2 阶段，而插件实际版本已达 v0.14.0，期间新增了大量重要功能（工作台、习惯打卡、四象限、番茄钟悬浮胶囊、手机端独立界面、专注工作台、项目工作台、AI 技能市场、Webhook 通知、企业微信机器人等），这些功能在 README 中完全没有体现。

## 目标

- **受众**：思源新用户 + 集市浏览者
- **核心任务**：让目标用户 10 秒内理解"这个插件能帮我解决什么"
- **成功标准**：新用户无需跳转文档即可了解核心价值；集市浏览者通过浏览就能决定是否尝试

## 设计方案

采用 **场景驱动型** 结构，从用户痛点出发组织内容，而非罗列功能列表。

### 整体结构

```
Hero 区（第一屏）：价值主张 + 核心差异 + 大图
  ├── 场景导航：7 个场景快速跳转
  ├── 场景 1：🗓️ 规划日程 → 日历 / 甘特图 / 四象限
  ├── 场景 2：🎯 专注执行 → 待办 + 番茄钟
  ├── 场景 3：✅ 习惯养成 → 习惯打卡 + 提醒
  ├── 场景 4：📁 项目管理 → 项目工作台（三栏）
  ├── 场景 5：🤖 AI 加持 → AI 对话 / 技能市场 / 企微
  ├── 场景 6：🖥️ 定制工作台 → 拖拽组合仪表盘
  ├── 场景 7：📱 移动办公 → 手机端独立界面
  ├── 快速开始（精简三步 + 标记速查表）
  ├── 安装指引（集市安装 / 手动安装）
  └── 文档 & 支持
```

### Hero 区

```markdown
# 任务助手

[release 徽章] | [更新日志] | [用户指南]

[English](README.md)

**在思源笔记里写任务，日历、甘特图、番茄钟自动呈现——文档即任务管理器。**

🔗  双向链接 — 点击任务跳转到笔记原文
🎯  文档即任务 — 写标记即可，无需切换应用
⚡  实时同步 — 改了笔记，所有视图同步更新
⏰  智能提醒 — 事项/习惯到点通知
```

### 场景导航

表格形式，每行包含：图标 + 场景名 + 一句话说明 + "适合你如果…"。

### 7 个场景

每个场景结构一致：
- 一句话场景标题
- 1-2 张核心截图
- 3-5 行要点说明（含版本号标注新功能引入版本）
- 亮点/子功能展开

### 快速开始

从现有的详细示例大幅精简为：
- 简洁的 markdown 示例（5 行）
- 标记速查表（📋、📅、✅/❌、⏰、🍅、🔁）
- 斜杠命令速查

### 安装

保持现有结构不变（集市 + 手动），文案精简。

### 文档 & 支持

保持现有链接列表，更新指向最新文档。
