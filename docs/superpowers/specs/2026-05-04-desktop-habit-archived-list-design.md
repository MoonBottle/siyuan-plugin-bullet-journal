# Desktop Habit Archived List Design

## 背景

当前习惯归档能力已经具备：

- 习惯定义行可持久化 `📦YYYY-MM-DD`
- 默认活跃列表会隐藏已归档习惯
- 已归档习惯详情可查看、不可打卡、可取消归档

但现在仍缺少一个明确的桌面端查看入口，导致用户归档后只能依赖外部导航或文档原文重新进入详情，无法在 HabitDock 内直接浏览已归档习惯。

## 目标

为桌面端 HabitDock 增加“查看已归档习惯”能力，满足以下目标：

- 默认活跃列表保持干净，不混入已归档习惯
- 在 Dock 内提供稳定、可发现的已归档列表入口
- 已归档列表与现有详情页复用同一套详情能力
- 从详情返回时，能回到进入详情前的列表上下文

## 非目标

- 不改移动端 Habit 面板
- 不把活跃列表和已归档列表混成一个列表
- 不在本次设计中加入批量取消归档、批量删除、批量导出
- 不修改归档的 Markdown 持久化模型

## 决策

采用 **顶栏独立图标入口 + 独立归档列表态**。

HabitDock 从当前“两态”扩展为“三态”：

1. `活跃列表`
2. `已归档列表`
3. `习惯详情`

默认进入 `活跃列表`。用户通过顶栏图标进入 `已归档列表`，点击归档习惯后进入现有 `习惯详情`。

## 交互设计

### 活跃列表态

- 保持当前默认行为：
  - 展示未归档习惯
  - 支持刷新
  - 支持点击进入详情
  - 支持文档入口与打卡入口
- 顶栏右上角增加“查看已归档习惯”图标按钮
  - 图标：`#iconInbox`
  - tooltip / aria-label：`查看已归档`

### 已归档列表态

- 只展示 `archivedAt` 存在的习惯
- 顶栏结构：
  - 左侧：返回按钮，返回到 `活跃列表`
  - 标题：`已归档习惯`
  - 右侧：刷新按钮
- 列表项主区域单击：
  - 进入该习惯的现有详情页
- 不提供今日打卡入口
  - 二元型打卡按钮不显示
  - 计数型 `+1` / 圆环打卡入口不显示
- 仍保留“打开文档”辅助入口

### 详情态

- 继续复用现有桌面端详情页
- 不区分活跃习惯详情和已归档习惯详情的承载组件
- 若当前习惯已归档：
  - 详情中继续显示“已归档”状态提示
  - 右上角继续保留“打开文档”
  - 右上角继续保留“取消归档”
  - 今日打卡入口保持禁用

## 返回规则

详情态的返回目标由“来源列表”决定：

- 从 `活跃列表` 进入详情，返回后回到 `活跃列表`
- 从 `已归档列表` 进入详情，返回后回到 `已归档列表`

这意味着详情不再只知道“是否选中了某个习惯”，还需要知道当前列表上下文。

## 状态模型

在桌面端习惯工作区增加列表模式状态：

```ts
type HabitListMode = 'active' | 'archived'
```

约束如下：

- `allHabits`：完整源数据，不做过滤
- `habits`：当前列表模式下展示的数据
  - `active` -> `!habit.archivedAt`
  - `archived` -> `Boolean(habit.archivedAt)`
- `selectedHabit`：始终从 `allHabits` 中按 `selectedHabitId` 查找

这样可以保证：

- 归档详情不会因为当前列表过滤而丢失
- 取消归档后详情仍能短暂稳定显示，直到用户返回列表

## 行为规则

### 列表切换

- 点击顶栏归档入口：
  - 切换到 `archived` 列表模式
  - 不自动选中第一个习惯
- 点击已归档列表返回按钮：
  - 切回 `active` 列表模式
  - 保持当前 `selectedDate`

### 取消归档后的表现

- 如果当前处于已归档详情并执行“取消归档”：
  - 详情允许继续显示当前习惯
  - 返回列表后，该习惯不再出现在已归档列表中
- 如果用户此时切回活跃列表，该习惯应出现在活跃列表中

### 刷新与外部数据变更

- 刷新逻辑继续沿用现有 `projectStore.refresh + DATA_REFRESH`
- 当前列表模式不因刷新被重置
- 若当前在 `archived` 列表且刷新后已无归档习惯：
  - 继续停留在 `已归档列表`
  - 显示空态

## UI 细节

### 顶栏图标

- 活跃列表态右上角顺序建议：
  1. 刷新
  2. 查看已归档
- 已归档列表态右上角：
  1. 刷新

所有新增图标都应补齐：

- `aria-label`
- `showIconTooltip`
- `hideIconTooltip`

### 已归档列表空态

当没有已归档习惯时，显示简洁空态：

- 标题：`暂无已归档习惯`
- 描述：`归档后的习惯会出现在这里`

不在本次设计中加入额外 CTA。

## 组件复用策略

- 尽量复用现有 `HabitWorkspaceListPane`
- 通过传参控制当前是否为归档列表态
- 归档列表态下关闭打卡按钮区，仅保留适合查看归档习惯的展示与跳转能力

不为归档列表单独复制一套列表组件，避免活跃列表和归档列表后续样式/行为漂移。

## 测试要求

### `useHabitWorkspace`

- 默认返回活跃习惯列表
- 切到 `archived` 模式后只返回已归档习惯
- `selectedHabit` 不受当前列表模式过滤影响
- 取消归档后，返回已归档列表时该习惯消失

### `DesktopHabitDock`

- 活跃列表顶栏展示“查看已归档”入口
- 点击后进入已归档列表态
- 已归档列表点击习惯进入详情
- 详情返回后回到已归档列表
- 已归档列表空态正常显示

## 验收标准

1. 默认 HabitDock 仍只展示活跃习惯
2. 用户可通过顶栏图标进入“已归档习惯”列表
3. 已归档列表只展示归档习惯，且不出现打卡按钮
4. 已归档列表单击习惯可进入现有详情页
5. 详情右上角仍可打开文档，并可取消归档
6. 从详情返回时，能回到进入详情前的列表上下文
