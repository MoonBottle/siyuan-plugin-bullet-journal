# 习惯打卡功能设计

## 一、功能概述

为任务助手插件增加习惯打卡功能，支持用户追踪日常习惯的执行情况。习惯是与任务同级的概念，打卡记录是习惯的子级。支持二元型（做了/没做）和计数型（带目标值+单位）两种习惯类型。

### 1.1 核心理念

- **与任务同级**：习惯和任务并列，都是项目下的一级概念
- **记录驱动**：习惯定义和打卡记录都在 Markdown 中，统计数据实时计算不持久化
- **用户驱动打卡**：频率仅用于衡量完成情况，系统不自动创建打卡记录，用户首次打卡时才创建
- **多 block 设计**：每次打卡是独立的 block，天然支持 SiYuan 的块操作和搜索

## 二、标记语法

### 2.1 习惯定义

```markdown
习惯名 🎯开始日期 [坚持N天] [目标值+单位] [⏰提醒时间] [🔄频率规则]
```

`🎯` 标记与 `#任务` 处于同一层级，写在习惯名之后。`🎯` 后接开始日期（必填），`坚持N天` 为习惯持续日历天数（可选，不设则无限期）。使用 `🔄` 作为习惯频率标记，与事项的 `🔁` 区分。

**核心规则**：
- `🎯` + 开始日期为必填项——习惯必须有明确的起始点
- 频率规则为必填项——习惯必须是周期性的
- `坚持N天` 表示从开始日期起的日历天数，到达后习惯结束（如 `🎯2026-04-01 坚持30天` → 4月30日结束）
- 频率仅表示"多久打卡一次"，不绑定具体日期——`🔄每周` 表示每周打卡一次（本周内任意一天即可），`🔄每周N天` 表示本周内完成 N 天即可
- 习惯和任务在文档中严格交替——按出现顺序切换上下文，遇到 `🎯` 切换为习惯上下文，遇到 `#任务` 切换为任务上下文

**二元型**（无目标值）：
```markdown
早起 🎯2026-04-01 坚持30天 🔄每天
冥想 🎯2026-04-01 🔄每天
周报 🎯2026-04-01 坚持84天 🔄每周五
```

**计数型**（有目标值+单位）：
```markdown
喝水 🎯2026-04-01 坚持21天 8杯 🔄每天
跑步 🎯2026-04-01 5公里 🔄每天
阅读 🎯2026-04-01 30分钟 🔄每天
背单词 🎯2026-04-01 坚持90天 50个 🔄每天
```

**带提醒**：
```markdown
喝水 🎯2026-04-01 坚持21天 8杯 ⏰09:00 🔄每天    ← 每天早上9点提醒喝水
早起 🎯2026-04-01 坚持30天 ⏰07:00 🔄每天          ← 每天早上7点提醒早起
```

支持的单位：次、杯、分钟、小时、公里、页、毫升、自定义。

### 2.2 频率规则

使用 `🔄` 作为习惯频率标记（简化版重复规则，与事项的 `🔁` 区分）：

| 中文格式 | 英文格式 | 说明 |
|----------|----------|------|
| `🔄每天` | `🔄daily` | 每天打卡 |
| `🔄每N天` | `🔄every N days` | 每N天打卡一次（如 `🔄每2天`） |
| `🔄每周` | `🔄weekly` | 每周打卡一次 |
| `🔄每周N天` | `🔄N days/week` | 每周打卡N天（如 `🔄每周3天`） |
| `🔄每周一三五` | `🔄Mon,Wed,Fri` | 每周指定周几打卡 |

> 与 `🔁` 的区别：`🔄` 是习惯专用频率标记，`🔁` 是事项重复标记。频率仅用于衡量完成情况（如本周应打卡N天），系统不会根据频率自动创建打卡记录。习惯的结束由 `坚持N天`（日历天数）决定，不支持 `🔚截止日期` 和 `🔢剩余次数`。

### 2.3 打卡记录

打卡记录由用户在首次打卡时创建（Dock UI 或笔记手写），位于习惯定义行下方。

**二元型**：
```markdown
早起 🎯2026-04-01 坚持30天 🔄每天    ← 习惯定义

早起 📅2026-04-06                     ← 用户打卡后创建
早起 📅2026-04-07                     ← 用户打卡后创建
```

**计数型**：
```markdown
喝水 🎯2026-04-01 坚持21天 8杯 🔄每天  ← 习惯定义

喝水 3/8杯 📅2026-04-07               ← 用户打卡记录进度（3/8）
喝水 8/8杯 📅2026-04-08               ← 达标，完成态由数值推导
```

**计数型记录格式为 `当前值/目标值+单位`**，目标值在创建 record 时从习惯定义复制过来，后续修改习惯目标不影响历史记录。例如习惯从 8杯改为 10杯后，旧记录仍显示 `3/8杯`，新记录显示 `0/10杯`。

### 2.4 标记顺序约定

```
习惯名 🎯开始日期 [坚持N天] [目标值+单位] [⏰提醒时间] [🔄频率规则]
```

### 2.5 解析规则

| Markdown | 解析结果 |
|----------|----------|
| `早起 🎯2026-04-01 坚持30天 🔄每天` | `Habit { type: 'binary', startDate: '2026-04-01', durationDays: 30, endDate: '2026-04-30', frequency: { type: 'daily' } }` |
| `喝水 🎯2026-04-01 坚持21天 8杯 🔄每天` | `Habit { type: 'count', startDate: '2026-04-01', durationDays: 21, endDate: '2026-04-21', target: 8, unit: '杯', frequency: { type: 'daily' } }` |
| `跑步 🎯2026-04-01 5公里 🔄每2天` | `Habit { type: 'count', startDate: '2026-04-01', target: 5, unit: '公里', frequency: { type: 'every_n_days', interval: 2 } }` |
| `早起 📅2026-04-06` | `CheckInRecord { content: '早起', date: '2026-04-06' }` |
| `喝水 3/8杯 📅2026-04-06` | `CheckInRecord { content: '喝水', currentValue: 3, targetValue: 8, unit: '杯' }` |
| `喝水 8/8杯 📅2026-04-06` | `CheckInRecord { content: '喝水', currentValue: 8, targetValue: 8, unit: '杯' }` |

## 三、数据模型

### 3.1 Habit 接口

```typescript
interface Habit {
  name: string                  // 习惯名（如"喝水"、"早起"）
  docId: string                 // 所属文档 ID
  blockId: string               // SiYuan block ID，作为唯一标识
  lastBlockId?: string          // 最后一个 record 的 block ID（用于插入位置）
  type: 'binary' | 'count'     // 二元型 / 计数型
  startDate: string             // 开始日期（YYYY-MM-DD，必填）
  durationDays?: number         // 持续日历天数（可选，如30天），到达后习惯结束
  endDate?: string              // 计算字段：startDate + durationDays - 1（仅 durationDays 存在时）
  target?: number               // 目标值（计数型，如 8）
  unit?: string                 // 单位（计数型，如"杯"）
  frequency?: HabitFrequency    // 频率规则（简化版，必填）
  reminder?: ReminderConfig     // 提醒配置（可选，复用已有）
  records: CheckInRecord[]      // 打卡记录
  links?: Link[]                // 链接
  pomodoros?: PomodoroRecord[]  // 番茄钟记录
}

type HabitFrequency = {
  type: 'daily' | 'every_n_days' | 'weekly' | 'n_per_week' | 'weekly_days'
  interval?: number             // 每 N 天的间隔（如 2 = 每2天）
  daysPerWeek?: number          // 每周 N 天（如 3 = 每周3天）
  daysOfWeek?: number[]         // 每周指定周几（0=周日, 1=周一, ...）
}
```

### 3.2 CheckInRecord 接口

```typescript
interface CheckInRecord {
  content: string               // 打卡日志内容（默认等于习惯名，用户可自定义修改）
  date: string                  // YYYY-MM-DD
  docId: string
  blockId: string               // SiYuan block ID，作为唯一标识

  // 计数型专用
  currentValue?: number         // 当前值（如 3）
  targetValue?: number          // 目标值（如 8）
  unit?: string                 // 单位

  // 所属习惯引用
  habitId: string               // 所属习惯的 blockId
}
```

### 3.3 Project 接口扩展

```typescript
interface Project {
  // ... 现有字段
  tasks: Task[]                 // 已有
  habits: Habit[]               // 新增
}
```

### 3.4 统计模型（纯计算，不持久化）

```typescript
interface HabitStats {
  habitId: string
  // 打卡次数
  monthlyCheckins: number       // 本月打卡次数（达标天数）
  totalCheckins: number         // 总打卡次数（达标天数）
  // 连续性（按频率周期计算）
  currentStreak: number         // 当前连续：每天→天数，每N天→周期数，每周/每周N天/指定周几→周数
  longestStreak: number         // 最长连续（同上按周期计算）
  // 完成率
  completionRate: number        // 总完成率 (0-1)
  monthlyCompletionRate: number // 本月完成率 (0-1)
  weeklyCompletionRate: number  // 本周完成率 (0-1)
  // 计数型专用
  totalValue?: number           // 累计值（如总共跑了 120 公里）
  averageValue?: number         // 日均值
  // 进度
  isCompleted: boolean          // 习惯是否已结束（durationDays 到期）
  isPeriodCompleted: boolean    // 当期是否已达标（如本周/本月已打卡）
}
```

## 四、斜杠命令

适配习惯打卡功能的斜杠命令，在编辑器中输入 `/` 唤起：

### 习惯标记命令

| 命令 | 触发词 | 功能说明 |
|------|--------|----------|
| 创建/编辑习惯 | `/xg`, `/habit` | 非习惯行：打开创建弹框；已有习惯行：打开编辑弹框 |
| 打卡 | `/dk`, `/checkin` | 根据当前行类型执行不同操作（见下表） |
| 打开习惯 Dock | `/xgd`, `/habits` | 打开习惯打卡侧边栏 |

**创建习惯弹框 UI**：

`/xg` 弹框包含以下字段：
- 习惯名（文本输入）
- 开始日期（日期选择器，默认今天）
- 坚持天数（数字输入，可选）
- 类型切换（二元型 / 计数型）
- 目标值+单位（计数型时显示）
- 提醒时间（可选，复用 ReminderSettingDialog）
- 频率（选择：每天 / 每N天 / 每周 / 每周N天 / 指定周几）

**创建模式**（非习惯行触发）：确认后在当前光标位置插入完整的习惯定义行。
**编辑模式**（已有习惯行触发）：弹框预填当前值，确认后更新习惯定义行。
```markdown
喝水 🎯2026-04-01 坚持21天 8杯 ⏰09:00 🔄每天
```

**使用示例**：

```markdown
# 输入斜杠命令打开创建弹框
/xg
# 弹框填写后自动插入：喝水 🎯2026-04-01 坚持21天 8杯 🔄每天
```

**`/dk` 打卡命令行为**：

| 触发行类型 | 行为 |
|-----------|------|
| 习惯定义行（今日未打卡） | 今日打卡一次：二元型直接创建 record，计数型当前值 +1（无 record 则创建） |
| 习惯定义行（今日已打卡） | 提示「今日已打卡」 |
| 今日打卡记录行（二元型） | 已打卡，提示「今日已打卡」 |
| 今日打卡记录行（计数型） | 当前值 +1，是否完成由 `currentValue >= targetValue` 推导 |
| 历史打卡记录行 | 打开习惯 Dock，定位到该习惯详情 |

## 五、解析流程

### 5.1 整体流程

```
笔记 Markdown (Kramdown)
  → core.ts 遍历 block 层级
  → 识别 🎯 行为习惯定义（类似识别 #任务）
  → habitParser.ts 解析习惯属性（类型、目标值、单位、频率规则）
  → 习惯下方的 📅日期行 → 解析为 CheckInRecord（类似事项解析）
  → projectStore.habits[] 存储
```

### 5.2 core.ts 解析逻辑扩展

```
遍历 block 层级：
  ## 项目名          → project
  #任务 / 📋        → task 开始
  🎯                → habit 开始
  下方带 📅日期的行  → 归属当前 task 的 item，或 habit 的 record
```

`🎯` 行和 `#任务` 行处于同一个层级，下方的 `📅日期` 行归属最近的 `🎯` 或 `#任务`。

### 5.3 habitParser.ts

负责解析习惯定义行：

- 识别 `🎯` 标记（后缀风格，在习惯名之后），提取习惯名
- 解析 `🎯` 后的开始日期（必填）
- 解析 `坚持N天` 为 `streakGoal`（可选）
- 解析目标值+单位（如 `8杯`、`5公里`、`30分钟`）
- 无目标值 → `type: 'binary'`
- 有目标值 → `type: 'count'`，填充 `target` 和 `unit`
- 解析 `🔄` 频率规则（简化版，不支持截止日期和剩余次数）

### 5.4 打卡记录解析

习惯下方的 `📅日期` 行解析为 `CheckInRecord`：

- 提取日期（复用现有日期解析逻辑，支持 `@` 和 `📅`）
- 计数型：从内容中提取 `N/M单位` 格式的当前值和目标值
- 完成态不再写入 `✅`，统一由 record 存在性或计数值是否达标推导
- 关联到所属习惯的 `habitId`

## 六、打卡流程

### 6.1 打卡记录创建（用户驱动）

系统**不自动创建**打卡记录。打卡记录在用户**首次打卡时**创建。

**触发时机**：
- 用户在 Dock UI 点击打卡按钮
- 用户在笔记中手写打卡记录行

**创建规则**：
```
1. 用户点击打卡 → 检查今天是否已有 record
2. 如果没有 → 创建一个新的 record block（二元型写日期记录，计数型记录初始值）
3. 如果已有 → 更新已有 record 的值
4. 位置：在最后一个 record 后面（或习惯定义行后面）
```

**首次打卡生成的 record block**：

| 类型 | 用户操作 | 生成的 Markdown |
|------|----------|-----------------|
| 二元型 | 点击打卡 | `习惯名 📅2026-04-07` |
| 计数型 | 点击 +1 或输入值 | `习惯名 1/8杯 📅2026-04-07` |

如果习惯定义有 `⏰` 提醒，系统根据习惯定义 + 频率直接计算提醒时间，不依赖 record block。
计数型 record 的目标值在创建时从习惯定义复制，后续修改习惯目标不影响已有记录。

### 6.2 Dock 打卡操作

Dock UI 是打卡的主入口：

**二元型**：
- 今日未打卡 → 显示「打卡」按钮，点击创建 record
- 今日已打卡 → 显示「已打卡」状态（仍显示，不隐藏）

**计数型**：
- 显示 `+1` 按钮（点击快速递增）和自定义输入（长按/右键弹出输入框直接填写具体值）
- 首次点击 `+1` 时创建 record（`1/目标值单位`），后续点击累加
- 也可直接输入具体值
- 达标（`currentValue >= targetValue`）不改写文案，仅更新完成态展示

**频率与完成率**：
- 频率规则用于计算「本周应打卡N天，已打卡M天」
- 不影响 record 创建，纯粹是统计维度的参考
- 已满足当期要求的习惯仍显示在 Dock 中，标记为「本周已达标」（灰色/淡化）

### 6.3 补打卡

支持为历史日期补打卡：

- **Dock 中切换日期**：习惯 Dock 的周日期行支持切换查看历史日期，点击对应日期后可为该日期打卡
- **补打卡操作**：与当日打卡操作一致，创建的 record 带对应历史日期（如 `早起 📅2026-04-05`）
- **插入位置**：补打卡的 record 按日期顺序插入到习惯定义下方的合适位置
- **streak 影响**：补打卡会影响连续天数计算——如果补打卡填上了之前的空缺，streak 可能恢复

### 6.4 编辑与撤销

**编辑打卡记录**：
- 计数型 record：可通过 Dock UI 修改已有记录的值（如把 3/8杯 改为 5/8杯），更新对应 block 的 Kramdown
- 修改后自动重新判断是否达标，更新 UI 完成态

**撤销打卡**：
- 支持撤销当天或历史日期的打卡，删除对应的 record block
- Dock UI 中提供「撤销打卡」按钮（如右键菜单或滑动操作）

### 6.5 笔记中直接编辑

用户也可以在笔记中手写打卡记录行，解析器会识别并纳入统计。

**二元型**：直接写 `习惯名 📅2026-04-07`

**计数型**：写 `习惯名 N/目标值单位 📅2026-04-07`，完成态由数值推导

**计数型打卡值的更新方式**：

| 方式 | 操作 | 示例 |
|------|------|------|
| 直接编辑 | 修改 Markdown 中的数值 | `喝水 3/8杯` → `喝水 8/8杯` |
| Dock UI +1 | 点击 +1 按钮 | 3杯 → 4杯，自动更新 block |
| Dock UI 设定值 | 输入框输入具体值 | 直接改为 8杯 |

### 6.6 提醒机制

习惯提醒**不依赖 record block**，直接从习惯定义计算。

**计算逻辑**：
```
对每个有 ⏰ 的 habit：
  1. 先判断今天是否仍在习惯有效期内
     - 未到 startDate → 不提醒
     - 超过 endDate（如有）→ 不提醒
  2. 根据频率判断今天是否需要提醒
     - 🔄每天 → 每天提醒，今日已打卡则跳过
     - 🔄每2天 → 从 startDate 起命中周期日才提醒
     - 🔄每周 → 当周未达标前，每天提醒；当周完成 1 次后停止提醒
     - 🔄每周3天 → 当周未达标前，每天提醒；当周完成 3 天后停止提醒
     - 🔄每周一三五 → 仅周一/三/五提醒；当天已打卡则跳过
  3. 计算提醒时间
     - 绝对时间：⏰09:00 → 今天 09:00
     - 已过了提醒时间但当前仍未达标 → 仍然提醒（可配置）
```

**与现有 reminderService 的区别**：
- 现有：提醒绑定在 Item block 上，触发时读取 item.reminder
- 习惯：提醒由 habit + frequency 动态计算，不存储在 record 中
- 需要扩展 reminderService 或新增 habitReminder 逻辑

## 七、统计计算

### 7.1 连续天数（Streak）

Streak 按习惯频率对应的日历周期计算：

| 频率 | Streak 单位 | 计算逻辑 |
|------|------------|----------|
| `🔄每天` | 天 | 从今天往前每天检查，达标+1，否则中断 |
| `🔄每N天` | 周期 | 从当前周期往前检查，每个N天周期内至少打卡1次则+1 |
| `🔄每周` | 周 | 从本周往前检查，每个自然周（周一到周日）内至少打卡1次则+1 |
| `🔄每周N天` | 周 | 从本周往前检查，每个自然周内打卡天数 ≥ N 则+1 |
```
对于 🔄每天 的习惯：
  从今天往前遍历 records：
    二元型：record 存在 → 连续+1，否则中断
    计数型：currentValue >= targetValue → 连续+1，否则中断
    遇到第一个非完成/非达标 → 停止

对于 🔄每周 / 🔄每周N天 / 🔄每周一三五 的习惯：
  从本周往前遍历每个自然周：
    - 🔄每周：该周至少 1 天达标 → 连续+1，否则中断
    - 🔄每周N天：该周达标天数 >= N → 连续+1，否则中断
    - 🔄每周一三五：该周所有要求日都达标 → 连续+1，否则中断
```

**今日未打卡的特殊处理**：今天还没结束不算中断，`currentStreak` 从昨天开始算。

### 7.2 完成率

| 指标 | 计算方式 |
|------|----------|
| 总完成率 | `累计已完成要求数 / 累计应完成要求数`（按频率规则统计，不按自然天数平铺） |
| 本周完成率 | `本周已完成要求数 / 本周应完成要求数` |
| 本月完成率 | `本月已完成要求数 / 本月应完成要求数` |

计数型的"完成"：`currentValue >= targetValue` 算完成。
频率仅作为统计与提醒参考（如 `🔄每周3天` 表示本周应打卡3天），不控制 record 创建。

**分母定义**：
- `🔄每天`：按应完成天数计算
- `🔄每N天`：按应完成周期数计算
- `🔄每周`：按应完成周数计算（每周 1 次要求）
- `🔄每周N天`：按应完成天数计算，但统计窗口按自然周切分
- `🔄每周一三五`：按要求日总数计算

### 7.3 热力图

支持两种展示模式：

**按全部习惯聚合**（默认）：
```
对每个日期格子：
  颜色深度 = 当日达标习惯数 / 当日应有习惯数
  全完成 → 深绿 #52c41a
  部分完成 → 浅绿 #95de64
  未完成 → 灰 #e8e8e8
  非打卡日 → 留空
```

**按单个习惯**：
```
达标 → 深绿
部分完成（计数型未达标）→ 浅绿
未打卡 → 灰
```
