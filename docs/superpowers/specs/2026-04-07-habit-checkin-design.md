# 习惯打卡功能设计

## 一、功能概述

为任务助手插件增加习惯打卡功能，支持用户追踪日常习惯的执行情况。习惯是与任务同级的概念，打卡记录是习惯的子级。支持二元型（做了/没做）和计数型（带目标值+单位）两种习惯类型。

### 1.1 核心理念

- **与任务同级**：习惯和任务并列，都是项目下的一级概念
- **记录驱动**：习惯定义和打卡记录都在 Markdown 中，统计数据实时计算不持久化
- **用户驱动打卡**：频率仅用于衡量完成情况，系统不自动创建打卡记录，用户首次打卡时才创建
- **多 block 设计**：每次打卡是独立的 block，天然支持 SiYuan 的块操作和搜索

## 二、标记语法

### 2.1 习惯定义

```markdown
习惯名 🎯开始日期 [坚持N天] [目标值+单位] [⏰提醒时间] [🔄频率规则]
```

`🎯` 标记与 `#任务` 处于同一层级，写在习惯名之后。`🎯` 后接开始日期（必填），`坚持N天` 为连续打卡目标天数（可选，不设则无限期）。使用 `🔄` 作为习惯频率标记，与事项的 `🔁` 区分。

**核心规则**：
- `🎯` + 开始日期为必填项——习惯必须有明确的起始点
- 频率规则为必填项——习惯必须是周期性的
- 不支持截止日期和剩余次数——习惯是长期行为，没有"结束"概念

**二元型**（无目标值）：
```markdown
早起 🎯2026-04-01 坚持30天 🔄每天
冥想 🎯2026-04-01 🔄每天
周报 🎯2026-04-01 坚持12周 🔄每周五
```

**计数型**（有目标值+单位）：
```markdown
喝水 🎯2026-04-01 坚持21天 8杯 🔄每天
跑步 🎯2026-04-01 5公里 🔄每天
阅读 🎯2026-04-01 30分钟 🔄每天
背单词 🎯2026-04-01 坚持90天 50个 🔄每天
```

**带提醒**：
```markdown
喝水 🎯2026-04-01 坚持21天 8杯 ⏰09:00 🔄每天    ← 每天早上9点提醒喝水
早起 🎯2026-04-01 坚持30天 ⏰07:00 🔄每天          ← 每天早上7点提醒早起
```

支持的单位：次、杯、分钟、小时、公里、页、毫升、自定义。

### 2.2 频率规则

使用 `🔄` 作为习惯频率标记（简化版重复规则，与事项的 `🔁` 区分）：

| 中文格式 | 英文格式 | 说明 |
|----------|----------|------|
| `🔄每天` | `🔄daily` | 每天打卡 |
| `🔄每N天` | `🔄every N days` | 每N天打卡一次（如 `🔄每2天`） |
| `🔄每周` | `🔄weekly` | 每周打卡一次 |
| `🔄每周N天` | `🔄N days/week` | 每周打卡N天（如 `🔄每周3天`） |
| `🔄每周一三五` | `🔄Mon,Wed,Fri` | 每周指定周几打卡 |

> 与 `🔁` 的区别：`🔄` 不支持截止日期和剩余次数，习惯是长期行为。频率仅用于衡量完成情况（如本周应打卡N天），系统不会根据频率自动创建打卡记录。

### 2.3 打卡记录

打卡记录由用户在首次打卡时创建（Dock UI 或笔记手写），位于习惯定义行下方。

**二元型**：
```markdown
早起 🎯2026-04-01 坚持30天 🔄每天    ← 习惯定义

早起 📅2026-04-06 ✅                  ← 用户打卡后创建
早起 📅2026-04-07 ✅                  ← 用户打卡后创建
```

**计数型**：
```markdown
喝水 🎯2026-04-01 坚持21天 8杯 🔄每天  ← 习惯定义

喝水 3/8杯 📅2026-04-07               ← 用户打卡记录进度（3/8）
喝水 8/8杯 📅2026-04-08 ✅            ← 达标，自动标 ✅
```

**计数型记录格式为 `当前值/目标值+单位`**，目标值在创建 record 时从习惯定义复制过来，后续修改习惯目标不影响历史记录。例如习惯从 8杯改为 10杯后，旧记录仍显示 `3/8杯`，新记录显示 `0/10杯`。

### 2.4 标记顺序约定

```
习惯名 🎯开始日期 [坚持N天] [目标值+单位] [⏰提醒时间] [🔄频率规则]
```

### 2.5 解析规则

| Markdown | 解析结果 |
|----------|----------|
| `早起 🎯2026-04-01 坚持30天 🔄每天` | `Habit { type: 'binary', startDate: '2026-04-01', streakGoal: 30, frequency: { type: 'daily' } }` |
| `喝水 🎯2026-04-01 坚持21天 8杯 🔄每天` | `Habit { type: 'count', startDate: '2026-04-01', streakGoal: 21, target: 8, unit: '杯', frequency: { type: 'daily' } }` |
| `跑步 🎯2026-04-01 5公里 🔄每2天` | `Habit { type: 'count', startDate: '2026-04-01', target: 5, unit: '公里', frequency: { type: 'every_n_days', interval: 2 } }` |
| `早起 📅2026-04-06 ✅` | `CheckInRecord { content: '早起', date: '2026-04-06' }` |
| `喝水 3/8杯 📅2026-04-06` | `CheckInRecord { content: '喝水', currentValue: 3, targetValue: 8, unit: '杯' }` |
| `喝水 8/8杯 📅2026-04-06 ✅` | `CheckInRecord { content: '喝水', currentValue: 8, targetValue: 8, unit: '杯' }` |

## 三、数据模型

### 3.1 Habit 接口

```typescript
interface Habit {
  name: string                  // 习惯名（如"喝水"、"早起"）
  docId: string                 // 所属文档 ID
  blockId: string               // SiYuan block ID，作为唯一标识
  lastBlockId?: string          // 最后一个 record 的 block ID（用于插入位置）
  type: 'binary' | 'count'     // 二元型 / 计数型
  startDate: string             // 开始日期（YYYY-MM-DD，必填）
  streakGoal?: number           // 连续打卡目标天数（可选，如30天）
  target?: number               // 目标值（计数型，如 8）
  unit?: string                 // 单位（计数型，如"杯"）
  frequency?: HabitFrequency    // 频率规则（简化版，必填）
  reminder?: ReminderConfig     // 提醒配置（可选，复用已有）
  records: CheckInRecord[]      // 打卡记录
  links?: Link[]                // 链接
  pomodoros?: PomodoroRecord[]  // 番茄钟记录
}

type HabitFrequency = {
  type: 'daily' | 'every_n_days' | 'weekly' | 'n_per_week' | 'weekly_days'
  interval?: number             // 每 N 天的间隔（如 2 = 每2天）
  daysPerWeek?: number          // 每周 N 天（如 3 = 每周3天）
  daysOfWeek?: number[]         // 每周指定周几（0=周日, 1=周一, ...）
}
```

### 3.2 CheckInRecord 接口

```typescript
interface CheckInRecord {
  content: string               // 打卡日志内容（默认等于习惯名，用户可自定义修改）
  date: string                  // YYYY-MM-DD
  docId: string
  blockId: string               // SiYuan block ID，作为唯一标识

  // 计数型专用
  currentValue?: number         // 当前值（如 3）
  targetValue?: number          // 目标值（如 8）
  unit?: string                 // 单位

  // 所属习惯引用
  habitId: string               // 所属习惯的 blockId
}
```

### 3.3 Project 接口扩展

```typescript
interface Project {
  // ... 现有字段
  tasks: Task[]                 // 已有
  habits: Habit[]               // 新增
}
```

### 3.4 统计模型（纯计算，不持久化）

```typescript
interface HabitStats {
  habitId: string
  // 打卡次数
  monthlyCheckins: number       // 本月打卡次数（达标天数）
  totalCheckins: number         // 总打卡次数（达标天数）
  // 连续性
  currentStreak: number         // 当前连续天数
  longestStreak: number         // 最长连续天数
  // 完成率
  completionRate: number        // 总完成率 (0-1)
  monthlyCompletionRate: number // 本月完成率 (0-1)
  // 计数型专用
  totalValue?: number           // 累计值（如总共跑了 120 公里）
  averageValue?: number         // 日均值
}
```

## 四、斜杠命令

适配习惯打卡功能的斜杠命令，在编辑器中输入 `/` 唤起：

### 习惯标记命令

| 命令 | 触发词 | 功能说明 |
|------|--------|----------|
| 创建/编辑习惯 | `/xg`, `/habit` | 非习惯行：打开创建弹框；已有习惯行：打开编辑弹框 |
| 打卡 | `/dk`, `/checkin` | 为当前打卡记录行添加 `✅`（二元型）或弹出计数输入（计数型） |
| 打开习惯 Dock | `/xgd`, `/habits` | 打开习惯打卡侧边栏 |

**创建习惯弹框 UI**：

`/xg` 弹框包含以下字段：
- 习惯名（文本输入）
- 开始日期（日期选择器，默认今天）
- 坚持天数（数字输入，可选）
- 类型切换（二元型 / 计数型）
- 目标值+单位（计数型时显示）
- 提醒时间（可选，复用 ReminderSettingDialog）
- 频率（选择：每天 / 每N天 / 每周 / 每周N天 / 指定周几）

**创建模式**（非习惯行触发）：确认后在当前光标位置插入完整的习惯定义行。
**编辑模式**（已有习惯行触发）：弹框预填当前值，确认后更新习惯定义行。
```markdown
喝水 🎯2026-04-01 坚持21天 8杯 ⏰09:00 🔄每天
```

**使用示例**：

```markdown
# 输入斜杠命令打开创建弹框
/xg
# 弹框填写后自动插入：喝水 🎯2026-04-01 坚持21天 8杯 🔄每天

# 输入斜杠命令快速打卡
早起 📅2026-04-06 /dk
# 结果：早起 📅2026-04-06 ✅
```

## 五、解析流程

### 5.1 整体流程

```
笔记 Markdown (Kramdown)
  → core.ts 遍历 block 层级
  → 识别 🎯 行为习惯定义（类似识别 #任务）
  → habitParser.ts 解析习惯属性（类型、目标值、单位、频率规则）
  → 习惯下方的 📅日期行 → 解析为 CheckInRecord（类似事项解析）
  → projectStore.habits[] 存储
```

### 5.2 core.ts 解析逻辑扩展

```
遍历 block 层级：
  ## 项目名          → project
  #任务 / 📋        → task 开始
  🎯                → habit 开始
  下方带 📅日期的行  → 归属当前 task 的 item，或 habit 的 record
```

`🎯` 行和 `#任务` 行处于同一个层级，下方的 `📅日期` 行归属最近的 `🎯` 或 `#任务`。

### 5.3 habitParser.ts

负责解析习惯定义行：

- 识别 `🎯` 标记（后缀风格，在习惯名之后），提取习惯名
- 解析 `🎯` 后的开始日期（必填）
- 解析 `坚持N天` 为 `streakGoal`（可选）
- 解析目标值+单位（如 `8杯`、`5公里`、`30分钟`）
- 无目标值 → `type: 'binary'`
- 有目标值 → `type: 'count'`，填充 `target` 和 `unit`
- 解析 `🔄` 频率规则（简化版，不支持截止日期和剩余次数）

### 5.4 打卡记录解析

习惯下方的 `📅日期` 行解析为 `CheckInRecord`：

- 提取日期（复用现有日期解析逻辑，支持 `@` 和 `📅`）
- 计数型：从内容中提取 `N/M单位` 格式的当前值和目标值
- `✅` 标记：计数型达标时自动追加，二元型打卡时自动追加
- 关联到所属习惯的 `habitId`

## 六、打卡流程

### 6.1 打卡记录创建（用户驱动）

系统**不自动创建**打卡记录。打卡记录在用户**首次打卡时**创建。

**触发时机**：
- 用户在 Dock UI 点击打卡按钮
- 用户在笔记中手写打卡记录行

**创建规则**：
```
1. 用户点击打卡 → 检查今天是否已有 record
2. 如果没有 → 创建一个新的 record block（二元型标记 `✅`，计数型记录初始值）
3. 如果已有 → 更新已有 record 的值
4. 位置：在最后一个 record 后面（或习惯定义行后面）
```

**首次打卡生成的 record block**：

| 类型 | 用户操作 | 生成的 Markdown |
|------|----------|-----------------|
| 二元型 | 点击打卡 | `习惯名 📅2026-04-07 ✅` |
| 计数型 | 点击 +1 或输入值 | `习惯名 1/8杯 📅2026-04-07` |

如果习惯定义有 `⏰` 提醒，系统根据习惯定义 + 频率直接计算提醒时间，不依赖 record block。
计数型 record 的目标值在创建时从习惯定义复制，后续修改习惯目标不影响已有记录。

### 6.2 Dock 打卡操作

Dock UI 是打卡的主入口：

**二元型**：
- 今日未打卡 → 显示「打卡」按钮，点击创建 record 并标记 `✅`
- 今日已打卡 → 显示「已打卡 ✅」状态

**计数型**：
- 显示 `+1` 按钮和自定义输入
- 首次点击 `+1` 时创建 record（`1/目标值单位`），后续点击累加
- 也可直接输入具体值
- 达标（`currentValue >= targetValue`）自动追加 `✅`

**频率与完成率**：
- 频率规则用于计算「本周应打卡N天，已打卡M天」
- 不影响 record 创建，纯粹是统计维度的参考

### 6.3 笔记中直接编辑

用户也可以在笔记中手写打卡记录行，解析器会识别并纳入统计。

**二元型**：直接写 `习惯名 📅2026-04-07 ✅`

**计数型**：写 `习惯名 N/目标值单位 📅2026-04-07`，达标时追加 `✅`

**计数型打卡值的更新方式**：

| 方式 | 操作 | 示例 |
|------|------|------|
| 直接编辑 | 修改 Markdown 中的数值 | `喝水 3/8杯` → `喝水 8/8杯` |
| Dock UI +1 | 点击 +1 按钮 | 3杯 → 4杯，自动更新 block |
| Dock UI 设定值 | 输入框输入具体值 | 直接改为 8杯 |

### 6.4 提醒机制

习惯提醒**不依赖 record block**，直接从习惯定义计算。

**计算逻辑**：
```
对每个有 ⏰ 的 habit：
  1. 根据频率判断今天是否应为打卡日
     - 🔄每天 → 每天都是
     - 🔄每2天 → 从 startDate 起，每隔2天
     - 🔄每周 → 本周应打卡1天
     - 🔄每周3天 → 本周应打卡3天
     - 🔄每周一三五 → 本周1/3/5应打卡
  2. 如果今天是打卡日 → 计算提醒时间
     - 绝对时间：⏰09:00 → 今天 09:00
     - 已过了提醒时间但今天还没打卡 → 仍然提醒（可配置）
  3. 今天已打卡 → 跳过提醒
```

**与现有 reminderService 的区别**：
- 现有：提醒绑定在 Item block 上，触发时读取 item.reminder
- 习惯：提醒由 habit + frequency 动态计算，不存储在 record 中
- 需要扩展 reminderService 或新增 habitReminder 逻辑

## 七、统计计算

### 7.1 连续天数（Streak）

```
从今天往前遍历 records：
  二元型：record 存在 → 连续+1，否则中断
  计数型：currentValue >= targetValue → 连续+1，否则中断
  遇到第一个非完成/非达标 → 停止
```

**今日未打卡的特殊处理**：今天还没结束不算中断，`currentStreak` 从昨天开始算。

### 7.2 完成率

| 指标 | 计算方式 |
|------|----------|
| 总完成率 | `已完成天数 / 总天数`（从 startDate 至今） |
| 本周完成率 | `本周已完成天数 / 频率要求天数`（根据 🔄 频率计算本周应打卡天数） |
| 本月完成率 | 同上，按月算 |

计数型的"完成"：`currentValue >= targetValue` 算完成。
频率仅作为统计参考（如 `🔄每周3天` 表示本周应打卡3天），不控制 record 创建。

### 7.3 热力图

支持两种展示模式：

**按全部习惯聚合**（默认）：
```
对每个日期格子：
  颜色深度 = 当日达标习惯数 / 当日应有习惯数
  全完成 → 深绿 #52c41a
  部分完成 → 浅绿 #95de64
  未完成 → 灰 #e8e8e8
  非打卡日 → 留空
```

**按单个习惯**：
```
达标 → 深绿
部分完成（计数型未达标）→ 浅绿
未打卡 → 灰
```

## 八、UI 设计

习惯打卡以 **Dock** 形式存在（类似 TodoDock），桌面端和移动端均支持，布局各异。

### 8.1 注册方式

在 `src/index.ts` 的 `registerDocks()` 中注册新 Dock，与 TodoDock 同级：

```typescript
this.addDock({
  config: {
    position: 'RightBottom',
    size: { width: 320, height: 400 },
    icon: 'iconCheck',     // 或其他合适的图标
    title: t('habit').title
  },
  data: {},
  type: DOCK_TYPES.HABIT,
  init() { /* 同 TodoDock 模式 */ },
  destroy() { /* 同 TodoDock 模式 */ }
});
```

### 8.2 桌面端 Dock 布局

参考 TodoDock 的结构模式，采用折叠式两层设计：

**第一层（默认展示）**：周日期 + 习惯列表

```
┌─────────────────────────────────────────────────┐
│  🎯 习惯打卡                                    │  ← block__icons 标题栏
├─────────────────────────────────────────────────┤
│  一    二    三    四    五    六    日           │  ← 周日期行（周一到周日，当天高亮）
│ 4/7   4/8   4/9   4/10  4/11  4/12  4/13       │
├─────────────────────────────────────────────────┤
│  💧 喝水   5/8杯    [·] ●●●                     │  ← 计数型：进度点 + 当前值
│  ✅ 早起   已打卡   ●●●●●●●                     │  ← 二元型：打卡状态 + 连续天数
│  🏃 跑步   3/5公里  [·] ●●●                     │
│  🧘 冥想   待打卡                               │
│  📖 阅读   0/30分钟  [·]                        │
└─────────────────────────────────────────────────┘
```

- **周日期行**：显示本周周一到周日，当天日期高亮，点击可切换查看某天的习惯
- **习惯列表**：每个习惯一行，显示名称 + 进度/状态
- **进度指示**：
  - 二元型：`已打卡` / `待打卡` + 连续天数点阵
  - 计数型：`当前值/目标值+单位` + 进度点阵
- **快捷操作**：点击习惯行展开第二层

**第二层（展开详情）**：点击习惯后展开该习惯的统计卡片、打卡日历和打卡日志

```

│  ← 💧 喝水  8杯/天 🔄每天                       │  ← 返回按钮 + 习惯信息

│  ┌──────────┐ ┌──────────┐                      │
│  │ 月打卡    │ │ 总打卡    │                      │  ← 统计卡片（2x2 网格）
│  │   8 次    │ │   42 次   │                      │     参考 DesktopTodoDock 风格
│  └──────────┘ └──────────┘                      │
│  ┌──────────┐ ┌──────────┐                      │
│  │ 月完成率  │ │ 连续打卡  │                      │

├─────────────────────────────────────────────────┤
│       2026年4月                                  │
│  一  二  三  四  五  六  日                       │  ← 打卡日历（月视图）

│   5   6   7   8   9  10  11                     │     深绿=达标，浅绿=部分，灰=未打卡
```

- **统计卡片**：四个卡片（月打卡次数、总打卡次数、月完成率、当前连续/最长连续天数），风格参考 DesktopTodoDock 的卡片组件
- **打卡日历**：月视图，用颜色深浅表示达标程度，可切换月份
- **打卡日志**：列表形式，解析规则与事项相同（日期、内容、状态），按时间倒序
- **计数型快捷操作**：日志中每条记录可点击 +1 或修改值

### 8.4 逻辑复用

**核心原则**：JS/TS 逻辑只写一套，桌面端和移动端各自写 UI，通过 composable/service 共享逻辑。

**共享逻辑层**（`src/services/` + `src/utils/` + `src/parser/`）：

| 模块 | 说明 |
|------|------|
| `habitService.ts` | 打卡创建 record、更新打卡值、达标判断 |
| `habitStatsUtils.ts` | 连续天数、完成率、热力图数据计算 |
| `habitParser.ts` | `🎯` 解析 + `🔄` 频率解析 + 打卡值解析 |
| `habitReminder.ts` | 习惯提醒计算逻辑 |

**桌面端 UI**（`src/components/habit/`）：
- 习惯列表、打卡月历、日志列表、统计卡片、打卡输入等组件

**移动端 UI**（`src/mobile/components/habit/`）：
- 独立的移动端 UI 组件，适配触屏交互和紧凑布局

**平台入口**：
- `src/tabs/HabitDock.vue` — 路由分发（同 TodoDock 模式）
- `src/tabs/DesktopHabitDock.vue` — 桌面端 Dock
- 移动端：集成在 `src/mobile/MobileTodoDock.vue` 中，通过底部导航切换显示
- `src/mobile/drawers/habit/HabitDetailDrawer.vue` — 移动端习惯详情抽屉

> UI 不共享，两端各自写。JS 逻辑通过 service/utils/composable 共享，不要写两遍。

## 十一、验收标准

- [ ] 可解析 `🎯` 标记定义习惯（二元型和计数型）
- [ ] 计数型支持目标值+单位（杯、公里、分钟等）
- [ ] 使用 `🔄` 简化频率规则（每天/每N天/每周/每周N天/指定周几）
- [ ] 打卡记录用户驱动创建（Dock UI 点击或笔记手写，系统不自动创建）
- [ ] 习惯定义可搭配 `⏰` 提醒，系统根据习惯+频率计算提醒时间
- [ ] 二元型打卡标记 `✅`
- [ ] 计数型达标自动标记 `✅`
- [ ] 计数型 UI 支持 +1 和自定义值
- [ ] 桌面端 Dock：周日期行 + 习惯列表
- [ ] 桌面端 Dock：点击习惯展开打卡日历 + 日志
- [ ] 移动端：集成在 MobileTodoDock 底部导航，通过打卡按钮切换显示
- [ ] 移动端：习惯详情抽屉（打卡日历 + 日志）
- [ ] 打卡日志解析规则与事项一致
- [ ] 统计：连续天数（当前+最长）
- [ ] 统计：完成率（总/本周/本月）
- [ ] 打卡月历颜色深浅表示达标程度
- [ ] 习惯与任务同级，不影响现有解析
- [ ] 斜杠命令：`/xg` 创建/编辑习惯弹框、`/dk` 快捷打卡、`/xgd` 打开习惯 Dock
