# 甘特图事项详情弹框修复

## 一、双弹框/多弹框问题（已修复）

用户反馈：点击甘特图中的事项时，会弹出多个「事项详情」弹框（2 个、4 个等），需多次关闭。问题非必现。

**根因**：`onTaskClick` 的 handler 被重复绑定。Tab 切换时，`destroy()` 仅执行 `innerHTML = ''`，可能未触发 Vue 的 `onUnmounted`，导致旧 handler 未解绑；再次进入甘特图时 `onMounted` 再次 attach，handler 累积。dhtmlx-gantt 支持同一事件绑定多个 handler，会全部执行。

**修复**：
1. **GanttView**：在 `attachEvent` 前先 `detachEvent` 已存在的 handler（利用模块级变量保存的 ID），确保同一时刻最多一个 handler
2. **dialog.ts**：`showEventDetailModal` 单例守卫，打开新弹框前关闭已存在的同类型弹框

## 三、动态导入导致的代码分割问题（已修复）

**问题**：使用 `await import()` 动态导入时，Vite 会生成代码分割的 chunk 文件（如 `recurringParser-xxx.js`），这些文件放在 `assets/` 目录下。但思源插件环境加载这些动态 chunk 时会报错：
```
Cannot find module './assets/recurringParser-xxx.js'
```

**根因**：思源插件的加载机制不支持动态导入产生的代码分割。

**修复**：将所有 `await import()` 改为静态 `import`：

| 文件 | 修改前 | 修改后 |
|------|--------|--------|
| `src/utils/dialog.ts` | `const { getNextOccurrenceDate } = await import('@/parser/recurringParser')` | `import { getNextOccurrenceDate, ... } from '@/parser/recurringParser'` |
| `src/utils/recurringService.ts` | `const { createNextOccurrence } = await import('@/services/recurringService')` | 移除（本来就是同文件） |

**验证**：构建后检查 `assets/` 目录，确认没有动态 chunk 文件生成，所有代码打包在 `index.js` 中。

## 五、弹框 ESC 键关闭问题（已修复）

**问题**：通过斜杠命令（`/tx`、`/cf`）打开的弹框，第一次按 ESC 键无法关闭，需要先点击弹框内部获取焦点。

**根因**：弹框打开时焦点仍在编辑器，不在弹框内，所以 ESC 键事件未被弹框捕获。

**修复**：在 `showReminderSettingDialog` 和 `showRecurringSettingDialog` 中，弹框创建后自动聚焦到内部第一个可交互元素（按钮/输入框等）：

```typescript
// 自动聚焦到弹框内，使 ESC 键立即生效
requestAnimationFrame(() => {
  const focusableEl = dialog.element.querySelector('button, input, [tabindex]:not([tabindex="-1"])') as HTMLElement;
  if (focusableEl) {
    focusableEl.focus();
  }
});
```

这样弹框打开后焦点立即在内部，ESC 键可以立即关闭弹框。

## 六、完成重复事项自动创建下一次（已完成）

**需求**：设置重复后，当完成事项时，应该自动创建下一次的重复事项。

**实现**：在以下三个地方的完成事项逻辑中添加自动创建下一次事项：

1. **TodoSidebar.vue** - Todo 面板中点击完成按钮
2. **GanttView.vue** - 甘特图中右键菜单完成
3. **CalendarView.vue** - 日历中右键菜单完成

**实现代码**：
```typescript
// 标记事项完成
const tag = getStatusTag('completed');
const success = await updateBlockContent(item.blockId, tag);

// 如果有重复规则，自动创建下一次事项
if (success && item.repeatRule && plugin) {
  await createNextOccurrence(plugin, item as Item);
}
```

**注意事项**：
- 只在成功标记完成且事项有 `repeatRule` 时才创建下一次
- 调用 `createNextOccurrence` 服务函数，它会处理结束条件（🔚日期、🔢次数）
- 如果已达到结束条件，则不会创建下一次

## 七、重复事项插入位置优化（已完成）

**问题**：自动创建的下次重复事项直接插在事项块下面，导致插入到事项相关子内容之前。

**修复**：新增 `getLastDescendantBlockId` 函数，查找当前事项块的最后一个后代块（包括子块、孙块等），然后在最后一个后代块之后插入新事项。

**实现逻辑**：
```typescript
// 获取当前事项块的最后一个后代块
const lastBlockId = await getLastDescendantBlockId(item.blockId);

// 在最后一个块后插入新 block
const result = await siyuanAPI.insertBlock(
  'markdown',
  newBlockContent,
  undefined,
  lastBlockId,  // 在最后一个后代块后插入
  undefined
);
```

**SQL 查询逻辑**：
- 通过 `path` 字段匹配子孙块（子块的 path 以 `当前块path + 当前块ID + '/'` 开头）
- 按 `path DESC, updated DESC` 排序，取最后一个
- 如果没有子块，则返回自身 ID

## 七、重复事项插入位置优化（已完成）

**问题**：自动创建的下次重复事项直接插在事项块下面，导致插入到事项相关子内容之前。

**修复**：结合 projectStore 中的事项列表，智能确定插入位置：

1. **获取同一文档中的所有事项**（通过 projectStore）
2. **按 lineNumber 排序**
3. **找到下一个事项**（lineNumber 更大的），插入到它前面
4. **如果没有下一个事项**，插入到文档末尾

**实现代码**：
```typescript
// 获取同一文档中的下一个事项
function getNextItemBlockId(item: Item): string | null {
  const pinia = getSharedPinia();
  if (!pinia) return null;

  const projectStore = useProjectStore(pinia);
  
  // 获取同一文档中的所有事项
  const docItems: Item[] = [];
  projectStore.projects.forEach(project => {
    project.tasks?.forEach(task => {
      task.items?.forEach(taskItem => {
        if (taskItem.docId === item.docId) {
          docItems.push(taskItem);
        }
      });
    });
  });

  // 按行号排序
  docItems.sort((a, b) => a.lineNumber - b.lineNumber);

  // 找到当前事项的索引
  const currentIndex = docItems.findIndex(i => i.blockId === item.blockId);
  if (currentIndex === -1) return null;

  // 获取下一个事项
  const nextItem = docItems[currentIndex + 1];
  return nextItem?.blockId || null;
}

// 插入逻辑
const nextItemBlockId = getNextItemBlockId(item);

if (nextItemBlockId) {
  // 有下一个事项，插入到它前面
  result = await siyuanAPI.insertBlock(
    'markdown',
    newBlockContent,
    nextItemBlockId,  // previousID 参数：插入到这个块前面
    undefined,
    undefined
  );
} else {
  // 没有下一个事项，插入到文档末尾
  const lastBlockId = await getDocLastBlockId(item.docId);
  // ...
}
```

**效果**：
- 新事项会插入在当前事项和下一个事项之间（平级）
- 不会插入到子内容中
- 如果没有后续事项，则插入到文档末尾

## 十、lastBlockId 解析修复（已完成）

**问题**：`lastBlockId` 只记录链接块，没有记录普通文本内容块。

**修复**：修改 `src/parser/core.ts`，在遇到下一个事项/任务之前，**每个块**都更新 `lastRelatedBlockIndex`：

```typescript
while (nextBlockIndex < blocks.length) {
  const nextBlock = blocks[nextBlockIndex];
  const nextContent = nextBlock.content.split('\n')[0].trim();

  if (isNextItemOrTaskLine(nextContent)) break;

  // 每个块都是当前事项的相关内容，更新最后一个相关块索引
  lastRelatedBlockIndex = nextBlockIndex;

  // ... 收集链接 ...
  nextBlockIndex++;
}
```

**关键改变**：
- 之前：只在找到链接时更新 `lastRelatedBlockIndex`
- 之后：**每个块**都更新 `lastRelatedBlockIndex`

**测试用例更新**：
1. 事项无相关内容
2. 事项有一个相关链接
3. 事项有普通文本内容（新增场景）
4. 事项有链接和普通内容混合
5. 多日期事项共享 lastBlockId
6. 事项是文档最后一个

**测试结果**：427 个测试通过
