## 🤔 什么是 MCP？

**MCP（Model Context Protocol）** 是由 Anthropic 推出的开放协议，旨在让 AI 助手能够安全、高效地访问本地数据。

### 为什么这很重要？

**以前的工作方式：**

你：打开思源笔记 → 切换到日历视图 → 查看任务 → 复制粘贴到 AI 对话框

AI：收到任务列表 → 分析 → 给出建议

**使用 MCP 后的工作方式：**

你：直接问 AI "我这周有哪些任务？"

AI：自动查询思源笔记 → 分析 → 给出建议

**节省 90% 的操作时间，让 AI 真正懂你！**

## 🚀 三步快速上手

### 第一步：更新插件

1. 打开思源笔记 → 设置 → 集市 → 已下载
2. 找到「子弹笔记助手」
3. 点击更新到 v0.7.0

### 第二步：获取 MCP 配置

1. 打开思源笔记设置 → 子弹笔记 → MCP 配置
2. 点击「复制 MCP 配置」按钮
3. 配置已自动复制到剪贴板

![image.png](https://b3logfile.com/file/2026/03/image-5S3GFsL.png)

### 第三步：配置 AI 助手

#### Trae 用户（推荐）

1. 打开 Trae → 设置 → MCP
2. 点击「添加 MCP 服务器」
3. 粘贴配置，将 `SIYUAN_TOKEN` 替换为你的 API Token
4. 保存并启用

![image.png](https://b3logfile.com/file/2026/03/image-PIgIg4w.png)

![image.png](https://b3logfile.com/file/2026/03/image-MCEm1w1.png)

配置 AI 智能体（可选但推荐）

为了让 AI 更好地理解如何使用子弹笔记数据，建议在 Trae 中配置智能体提示词：

1. 打开 Trae → 智能体 → 创建智能体
2. 填写智能体名称，如「子弹笔记助手」
3. 在提示词区域粘贴以下内容：
   
   你可以访问一个子弹笔记 MCP 服务器，该服务器提供以下工具：
   
   1. list_groups：列出所有项目分组。首先使用此工具了解项目组织结构。
   2. list_projects：列出所有项目，支持 groupId 过滤。每个项目包含 id、name、description、path、groupId 和 taskCount。
   3. filter_items：筛选子弹笔记事项，参数包括：
      - projectId/projectIds：按特定项目筛选
      - groupId：按项目分组筛选
      - startDate/endDate：按日期范围筛选（YYYY-MM-DD 格式）
      - status：按状态筛选（'pending' 待办、'completed' 已完成、'abandoned' 已放弃）
   
   何时使用这些工具：
   
   - 用户询问任务、项目或日程安排时
   - 用户想要追踪进度或回顾已完成的工作时
   - 用户需要规划或组织工作时
   - 用户要求汇总或报告子弹笔记数据时
   
   最佳实践：
   
   1. 始终先调用 list_groups 了解项目结构
   2. 使用 list_projects 获取所有项目的概览
   3. 使用 filter_items 配合适当的筛选条件获取具体任务事项
   4. 组合使用筛选条件进行精确查询（例如：查询本周某项目的待办事项）
4. 保存智能体配置

配置完成后，在与该智能体对话时，AI 会自动调用 MCP 工具获取你的子弹笔记数据。

## 🛠️ MCP 功能详解

### 三个核心工具

子弹笔记助手的 MCP 服务器为 AI 提供了三个数据访问工具：

#### 1️⃣ list_groups - 查询分组

**用途：** 获取所有项目分组信息

**返回示例：**

- 分组 ID: group-1，名称: 工作项目
- 分组 ID: group-2，名称: 个人学习
- 分组 ID: group-3，名称: 生活事务

#### 2️⃣ list_projects - 查询项目

**用途：** 获取所有项目列表，支持按分组筛选

**参数：**

- `groupId`（可选）：按分组 ID 筛选

**返回示例：**

- 项目 ID: doc-id-1
- 项目名称: MCP 功能开发
- 项目描述: 为插件添加 MCP 支持
- 项目路径: 工作/2026/MCP功能开发
- 所属分组: group-1
- 任务数量: 8

#### 3️⃣ filter_items - 筛选事项

**用途：** 精确查询任务事项，支持多种筛选条件

**参数：**

- `projectId` / `projectIds`：按项目筛选
- `groupId`：按分组筛选
- `startDate` / `endDate`：按日期范围筛选
- `status`：按状态筛选（pending/completed/abandoned）

**返回示例：**

- 事项 ID: item-1
- 内容: 完成 MCP 服务器开发
- 日期: 2026-03-05
- 状态: completed（已完成）
- 所属项目: MCP 功能开发
- 所属任务: 后端开发

## 📝 数据格式说明

### 任务标记规范

插件使用以下约定格式解析任务数据，请按此规则编写：

**示例格式：**

```
## 项目名称
> 项目描述
> [需求文档](http://doc.example.com)

任务名称 #任务 @L1
[任务链接](http://task.example.com)

具体工作项 @2026-03-05
另一个工作项 @2026-03-06 09:00:00~12:00:00
已完成工作项 @2026-03-04 #done

子任务名称 #任务 @L2
子任务工作项 @2026-03-07
```

### 标记说明

- `#任务` - 标记任务行，如 `开发功能 #任务`
- `@L1/@L2/@L3` - 任务层级，`@L1` 为父任务，`@L2` 为子任务
- `@YYYY-MM-DD` - 事项日期，如 `@2026-03-05`
- `@日期 时间~时间` - 带时间范围，如 `@2026-03-05 09:00:00~12:00:00`
- `#done` / `#已完成` - 已完成标记，如 `完成任务 #done`
- `#abandoned` / `#已放弃` - 已放弃标记，如 `取消任务 #abandoned`

## 💡 使用技巧

### 与 AI 对话的最佳实践

#### ✅ 明确时间范围

**好的提问：**

- "这周有哪些待办任务？"
- "下个月要完成什么？"
- "2026年第一季度有哪些项目？"

**避免模糊的提问：**

- ❌ "我有什么任务？"
- ❌ "接下来要做什么？"

#### ✅ 指定项目或分组

**好的提问：**

- "工作分组下有哪些项目？"
- "项目 A 还剩多少任务没完成？"
- "个人学习分组本周的进度如何？"

#### ✅ 组合筛选条件

**好的提问：**

- "工作项目中，本周待办的任务有哪些？"
- "上个月已完成的事项中，属于项目 A 的有哪些？"
- "项目 B 中延期超过 3 天的任务有哪些？"

## 🔒 隐私与安全

### 数据安全说明

- ✅ **本地运行**：MCP 服务器只在本地运行，数据不会上传到云端
- ✅ **本地通信**：AI 助手通过本地进程访问数据，无需网络传输
- ✅ **自主控制**：你可以随时启用或禁用 MCP 功能

### 安全建议

- ⚠️ 不要在任务内容中记录敏感信息（如密码、密钥等）
- ⚠️ 定期更新插件到最新版本，获取安全补丁
- ⚠️ 妥善保管 API Token，不要分享给他人

## ❓ 常见问题

### Q1: MCP 服务器启动失败？

**检查清单：**

1. ✅ 思源笔记已启动且插件已启用
2. ✅ API Token 正确配置（思源 → 设置 → 关于 → API Token）
3. ✅ 插件版本为 v0.6.0 或更高
4. ✅ 查看 AI 助手的 MCP 日志获取详细错误

### Q2: AI 无法读取数据？

**可能原因：**

1. 项目目录未配置 - 检查插件设置中的目录路径
2. 文件格式不正确 - 确保使用标准的子弹笔记格式
3. 任务标签不匹配 - 默认使用 `#任务` 标签

### Q3: 数据更新后 AI 看不到最新内容？

**解决方案：**

1. 在思源笔记中保存文件后，等待 1-2 秒
2. 重新发起对话，AI 会重新查询最新数据
3. 如果问题持续，尝试重启 AI 助手

### Q4: 支持哪些 AI 助手？

目前已测试支持：

- ✅ **Trae**（推荐，国内用户友好，可免费使用国产模型）

理论上其他支持 MCP 协议的 AI 助手（如 Cursor、Claude Desktop 等）也可以使用，但尚未经过充分测试。

**Trae 优势：**

- 国内网络环境友好，无需代理
- 免费使用国产大模型
- 内置 MCP 支持，配置简单
