## 🤔 什么是番茄工作法？

**番茄工作法**（Pomodoro Technique）是一种时间管理方法，由 Francesco Cirillo 于 1980 年代创立。

### 核心原理

1. **选择一个任务** - 明确当前要专注完成的工作
2. **设定计时器** - 通常为 25 分钟（一个番茄钟）
3. **专注工作** - 直到计时器响起，期间不做其他事情
4. **短暂休息** - 计时结束后休息 5 分钟
5. **循环往复** - 每 4 个番茄钟后，进行一次长休息（15-30 分钟）

### 为什么有效？

**传统工作方式：**

- 多任务并行，注意力分散
- 工作被打断后难以恢复专注
- 不清楚实际花费了多少时间
- 容易疲劳和倦怠

**番茄工作法：**

- 单任务专注，提高效率
- 明确的时间边界，减少干扰
- 可视化的时间记录，便于复盘
- 规律休息，保持精力充沛

**研究表明，使用番茄工作法可以提升 30% 以上的工作效率！**

## 🚀 快速上手

### 第一步：更新插件

1. 打开思源笔记 → 设置 → 集市 → 已下载
2. 找到「任务助手」

## 🛠️ 功能详解

### 1️⃣ 专注计时

**开始专注：**

- 从待办列表中选择事项（支持过期事项和今天事项）
- 快捷选择时长：15/25/45/60 分钟
- 或自定义输入 1-180 分钟的任意时长

**专注中：**

- 圆形进度条实时显示剩余时间
- 显示开始时间和预计结束时间
- 展示关联的项目、任务、事项信息
- 支持暂停/继续功能

**完成专注：**

- 倒计时结束自动完成
- 播放提示音提醒
- 显示系统级通知
- 自动在笔记中创建番茄钟记录

### 2️⃣ 番茄钟记录

**自动记录格式：**

```markdown
工作事项 @2026-03-10
🍅2026-03-10 15:45:32~15:50:32
```

**带暂停的记录格式：**

```markdown
工作事项 @2026-03-10
🍅42,2026-03-10 15:00:00~15:47:00
```

> 说明：数字 `42` 表示实际专注 42 分钟（不含暂停时间）

**暂停次数记录：**

当专注过程中使用了暂停功能，系统会记录暂停次数和实际专注时长：

```markdown
修复登录bug @2026-03-10
🍅42,2026-03-10 15:00:00~15:47:00
```

这表示：
计划专注时长：45分钟（从 15:00 到 15:47 共 47 分钟，含 2 分钟休息）
实际专注时长：42 分钟（扣除了暂停时间）
暂停次数：1 次（在后台状态中记录）

**手动添加记录：**

你也可以手动在笔记中添加番茄钟记录：

```markdown
🍅2026-03-10 15:00:00~15:25:00 专注描述
```

或使用带实际时长的格式（适用于有暂停的情况）：

```markdown
🍅20,2026-03-10 15:00:00~15:30:00 专注描述
```

### 3️⃣ 统计数据

**今日统计：**

- 今日番茄数：完成的番茄钟数量
- 今日专注时长：格式为 "2h 30m" 或 "45m"

**累计统计：**

- 总番茄数：所有时间累计的番茄数
- 总专注时长：累计专注时间

**记录列表：**

- 按日期分组展示历史记录
- 显示每个番茄钟的开始/结束时间
- 支持查看关联的事项详情

### 4️⃣ AI 智能助手

**支持的查询功能：**

- 查询今日/指定日期的番茄钟统计
- 获取番茄钟记录列表
- 生成专注度报告
- 分析专注趋势

**AI 工具调用：**

当询问专注相关问题时，AI 会自动调用以下工具：

- `get_pomodoro_stats` - 获取番茄钟统计数据
- `get_pomodoro_records` - 获取番茄钟详细记录
- `get_user_time` - 获取用户时间信息

**示例对话：**

```
用户：这周完成了几个番茄钟？

AI：我来帮您查看本周的番茄钟完成情况。

📊 本周专注统计（3月4日-3月10日）

🍅 番茄钟数量：18个
⏱️ 总专注时长：8小时 45分钟
📈 日均专注：1小时 15分钟

最专注的一天：3月6日（5个番茄钟）
```

### 5️⃣ 状态持久化

**专注状态自动保存：**

- 开始专注后，状态保存到本地文件
- 插件重启后能自动恢复专注状态
- 支持恢复暂停状态

**过期处理：**

- 如果专注已过期（超过设定时长），自动标记为完成
- 在文档中创建番茄钟记录

## 📝 使用技巧

### 选择合适的专注时长

| 任务类型 | 建议时长 | 说明 |
|---------|---------|------|
| 阅读/学习 | 25 分钟 | 保持注意力，避免疲劳 |
| 写作/创作 | 45 分钟 | 需要较长时间进入心流 |
| 编程开发 | 45-60 分钟 | 复杂任务需要连续思考 |
| 会议/沟通 | 15 分钟 | 快速处理沟通类任务 |

### 善用暂停功能

- 被打断时及时暂停，保证专注时间的准确性
- 暂停次数会统计，帮助你分析干扰频率
- 恢复后继续计时，无需重新开始

### 结合任务管理使用

- 在待办事项上右键选择「开始专注」
- 专注完成后，事项状态自动更新
- 通过 AI 助手查询每日番茄分布和专注统计

### 定期回顾统计

- 每日查看专注时长，了解工作效率
- 分析番茄分布，优化时间安排
- 长期追踪，培养专注习惯

## ❓ 常见问题

### Q1: 专注时切换标签页会影响计时吗？

**不会。** 番茄钟使用基于时间戳的计时方案，即使页面在后台运行，计时依然准确。当你切回页面时，时间会自动校准。

### Q2: 可以同时进行多个番茄钟吗？

**不可以。** 番茄工作法的核心是单任务专注。同一时间只能进行一个番茄钟，完成或取消后才能开始新的。

### Q3: 提前结束专注会记录吗？

**不会。** 提前结束（取消）专注不会创建番茄钟记录，只有正常完成的专注才会保存到笔记中。

### Q4: 如何手动添加番茄钟记录？

在事项下方直接添加以下格式的行：

```markdown
🍅2026-03-10 15:00:00~15:25:00 专注描述
```

或使用带实际时长的格式（适用于有暂停的情况）：

```markdown
🍅20,2026-03-10 15:00:00~15:30:00 专注描述
```

### Q6: AI 助手能查询哪些番茄钟数据？

AI 助手可以查询以下数据：

- **统计数据**：今日/指定日期的番茄数、专注时长
- **记录列表**：详细的番茄钟记录，包括时间、事项、项目
- **趋势分析**：周/月专注趋势，日均专注时长
- **项目统计**：按项目分组的专注时间

只需在 AI 对话框中自然语言提问，如：
- "今天专注度怎么样"
- "这周完成了几个番茄钟"
- "查看网站重构项目的专注时间"

### Q5: 专注完成没有收到通知？

请检查以下几点：

1. ✅ 浏览器/思源笔记的通知权限已开启
2. ✅ 系统音量未静音
3. ✅ 插件设置中的通知选项已启用
