# 产品概述

## 一、产品定位

任务助手是一款思源笔记插件，采用「记录驱动」的任务管理理念，让用户专注于记录已完成和待完成的事项，而非被提醒所打扰。

## 二、核心理念

### 2.1 记录驱动
- 专注于记录，而非提醒
- 用标记书写笔记，在视图中查看
- 无侵入式设计，使用标准 Markdown 格式

### 2.2 双向链接
- 点击视图中的任务可直接跳转到笔记对应位置
- 保持笔记与视图的实时同步

### 2.3 无专有格式锁定
- 使用标准 Markdown 语法
- 即使不使用插件，笔记内容依然可读
- 支持迁移到其他工具

## 四、用户价值

| 场景 | 价值 |
|------|------|
| 日常工作记录 | 快速记录任务和事项，无需切换应用 |
| 文档内查看 | 在笔记中右键或 Ctrl+点击即可打开事项详情 |
| 项目规划 | 通过甘特图可视化项目时间线 |
| 日程管理 | 日历视图直观展示工作安排 |
| 专注工作 | 番茄钟功能帮助保持专注 |
| 智能助手 | AI 对话查询任务、生成日报 |

## 五、目标用户

1. **知识工作者** - 需要记录和追踪工作任务
2. **项目经理** - 需要管理项目进度和时间线
3. **自由职业者** - 需要安排个人工作日程
4. **学生** - 需要管理学习任务和截止日期

## 六、使用流程

### 6.1 基本工作流

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  在笔记中     │────▶│  插件自动     │────▶│  在视图中     │
│  书写标记     │     │  解析数据     │     │  查看展示     │
└──────────────┘     └──────────────┘     └──────────────┘
```

### 6.2 详细流程

1. **创建项目文档** - 在思源笔记中创建 Markdown 文档
2. **编写任务格式** - 使用 `#任务` 标记任务，`@日期` 标记事项
3. **配置插件**（可选）- 设置扫描目录或分组
4. **查看视图** - 通过日历、甘特图、项目列表或待办 Dock 查看任务
5. **使用 AI 助手**（可选）- 查询任务、生成日报

## 七、核心概念

### 7.1 项目（Project）
- 对应一个思源笔记文档
- 包含多个任务
- 文档标题即为项目名称

### 7.2 任务（Task）
- 使用 `#任务` 或 `#task` 标记
- 包含层级信息（L1/L2/L3）
- 包含多个工作事项

### 7.3 事项（Item）
- 使用 `@日期` 标记
- 可设置时间范围
- 可标记状态（待办/已完成/已放弃）

### 7.4 番茄钟（Pomodoro）
- 使用 `🍅` 标记
- 记录专注时间段
- 支持手动记录和计时器自动记录

## 八、技术栈

- **前端框架**: Vue 3 + TypeScript
- **状态管理**: Pinia
- **UI 组件**: 思源主题组件
- **日历组件**: FullCalendar
- **甘特图组件**: dhtmlx-gantt
- **AI 集成**: OpenAI 兼容 API
- **MCP 服务**: Model Context Protocol
