## 解析器
- [x] `lineParser.ts` 中的正则表达式支持实际时长前缀
- [x] 正则表达式支持中英文逗号（`,` 或 `，`）作为分隔符
- [x] 正则表达式支持逗号后任意数量的空格（`\s*`）
- [x] `parsePomodoroLine()` 正确提取 `actualDurationMinutes`
- [x] 单元测试覆盖英文逗号格式（无空格）
- [x] 单元测试覆盖中文逗号格式
- [x] 单元测试覆盖逗号后1个空格
- [x] 单元测试覆盖逗号后多个空格
- [x] 单元测试覆盖不带实际时长的格式（向后兼容）
- [x] 所有现有测试仍然通过

## 统计计算
- [x] `getTodayFocusMinutes()` 优先使用 `actualDurationMinutes`
- [x] `getTotalFocusMinutes()` 优先使用 `actualDurationMinutes`
- [x] 无实际时长时使用 `durationMinutes` 计算

## 格式兼容性
- [x] 支持 `🍅5,2026-03-08 15:45:32~15:50:32 描述`（英文逗号，无空格）
- [x] 支持 `🍅5，2026-03-08 15:45:32~15:50:32 描述`（中文逗号）
- [x] 支持 `🍅5, 2026-03-08 15:45:32~15:50:32 描述`（逗号后1个空格）
- [x] 支持 `🍅5,   2026-03-08 15:45:32~15:50:32 描述`（逗号后多个空格）
- [x] 支持 `🍅5,2026-03-08 15:45:32 描述`（无结束时间有实际时长）
- [x] 支持 `🍅2026-03-08 15:45:32~15:50:32 描述`（向后兼容）
- [x] 支持 `🍅2026-03-08 15:45:32 描述`（向后兼容，默认25分钟）
