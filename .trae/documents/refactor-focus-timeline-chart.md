# FocusTimelineChart 重构计划

## 目标效果
1. 左侧 Y 轴显示时间标签：00:00, 04:00, 08:00, 12:00, 16:00, 20:00（24:00 存在但不显示）
2. 横向虚线分割线，与 Y 轴标签对齐（共 7 条线：00:00, 04:00, 08:00, 12:00, 16:00, 20:00, 24:00）
3. 中间热力图区域，显示每天 24 小时的专注时长
4. **底部 X 轴显示星期：一、二、三、四、五、六、日**
5. 时间线位置与 Y 轴标签正确对应（如 10:00 的数据在 08:00 和 12:00 之间）

## 当前问题
1. 虚线不显示
2. Y 轴标签与时间线单元格位置不对齐
3. 布局复杂，难以维护

## 重构方案

### 布局结构
使用 CSS Grid 布局替代 Flexbox，更精确控制行列对齐：

```
+------------------+
| 标题        控件  |
+------------------+
| 00:00 | 热力图   |
| 04:00 | 热力图   |
| 08:00 | 热力图   |
| 12:00 | 热力图   |
| 16:00 | 热力图   |
| 20:00 | 热力图   |
| (24)  |          |
+------------------+
|       | 一 二 三...|
+------------------+
```

### 具体实现
1. 使用 Grid 布局：
   - 8 行（6行时间刻度 + 1行24:00隐藏行 + 1行X轴标签）
   - 8 列（1列Y轴 + 7列天数据）
2. Y 轴标签和横向虚线在同一行
3. 每天的热力图使用 24 个单元格，按小时分布
4. **X轴标签放在最底部一行**
5. 统一高度计算，每 4 小时一个刻度区域

### 时间刻度对应关系
- 00:00 - 04:00：第 0-3 小时
- 04:00 - 08:00：第 4-7 小时
- 08:00 - 12:00：第 8-11 小时（10:00 在这里）
- 12:00 - 16:00：第 12-15 小时
- 16:00 - 20:00：第 16-19 小时
- 20:00 - 24:00：第 20-23 小时

1. 重写 template 结构，使用 Grid
2. 重写 CSS 样式
3. 保持现有的数据逻辑和交互
4. 简化 tooltip 实现
