# 日历事件样式修复计划

## 问题

修改弹框样式时，意外影响了日历中的事件样式。日历事件的颜色变成了不正常的颜色（看起来是灰色或浅色）。

## 问题分析

可能是修改 `index.scss` 时，新增的样式类名与日历组件的类名冲突，或者样式规则过于宽泛，影响到了日历事件。

需要检查：
1. `.sy-dialog-*` 类名是否与日历组件冲突
2. 是否有全局样式影响到了日历
3. 是否有颜色变量被覆盖

## 检查点

### 1. 检查 index.scss 中的样式

查看最近添加的样式是否使用了通用类名：
- `.sy-dialog-card`
- `.sy-dialog-icon`
- `.sy-dialog-copy-btn`
- 等等

### 2. 检查是否有全局样式

查看是否有不带作用域的样式规则，例如：
```scss
// 可能影响全局的样式
span { ... }
button { ... }
```

### 3. 检查颜色变量

查看是否修改了思源主题变量：
- `--b3-theme-primary`
- `--b3-theme-surface`
- `--b3-theme-background`
- 等等

## 修复方案

### 方案一：确保样式隔离

确保所有弹框样式都在 `.sy-dialog-content` 或特定容器内：

```scss
.sy-dialog-content {
  // 所有弹框样式放在这里
  .sy-dialog-card { ... }
  .sy-dialog-icon { ... }
}
```

### 方案二：检查冲突类名

如果 `.sy-dialog-*` 类名与日历组件冲突，考虑：
1. 重命名类名，使用更独特的前缀
2. 增加更具体的选择器

### 方案三：回滚检查

如果无法快速定位问题，可以：
1. 回滚 `index.scss` 到修改前的版本
2. 逐步添加样式，观察何时出现问题

## 实现步骤

1. **检查 `index.scss`** - 查看最近添加的样式
2. **添加作用域** - 确保样式不影响外部组件
3. **验证修复** - 检查日历事件颜色是否恢复正常

## 文件变更

| 文件 | 变更 |
|------|------|
| `src/index.scss` | 修复样式隔离，确保不影响日历组件 |

## 预期效果

- 弹框样式正常工作
- 日历事件颜色恢复正常
- 两者互不影响
