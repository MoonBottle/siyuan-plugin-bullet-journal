# 调整多日期测试文件计划

## 问题分析

根据 `data-format.md` 的规范，当前测试文件 `multi-date-test.md` 中很多事项是独立的，没有对应的任务。规范要求：

- **任务** (`#任务 @L1/@L2/@L3`)：代表目标或交付物，本身不带日期
- **事项** (`@日期`)：代表完成目标的具体行动，必须带日期，且应该属于某个任务

## 当前结构问题

当前测试文件中存在大量独立事项，例如：
```markdown
- 单日期事项A @2026-03-06
- 多日期英文A @2026-03-06, 2026-03-10, 2026-03-15
- 边界单时间 @2026-03-06 09:00:00
```

这些事项都没有对应的任务。

## 修改方案

将测试文件重新组织，为每个事项添加对应的任务。按测试类型分组：

### 1. 单日期事项测试
```markdown
## 单日期事项测试

- 单日期测试任务 #任务 @L1
  - 单日期事项A @2026-03-06
  - 单日期事项B @2026-03-10 14:00:00~15:00:00
  - 单日期事项C @2026-03-15 #done
  - 单日期事项D @2026-03-20 #abandoned
```

### 2. 多日期事项测试（英文逗号）
```markdown
## 多日期事项测试（英文逗号）

- 多日期英文测试任务 #任务 @L1
  - 多日期英文A @2026-03-06, 2026-03-10, 2026-03-15
  - 多日期英文B @2026-03-10 09:00:00~10:00:00, 2026-03-12 14:00:00~15:00:00
```

### 3. 多日期事项测试（中文逗号）
```markdown
## 多日期事项测试（中文逗号）

- 多日期中文测试任务 #任务 @L1
  - 多日期中文A @2026-03-06，2026-03-10，2026-03-15
  - 多日期中文B @2026-03-10 09:00:00~10:00:00，2026-03-12 14:00:00~15:00:00
```

### 4. 混合逗号测试
```markdown
## 多日期事项测试（中英文逗号混合）

- 多日期混合测试任务 #任务 @L1
  - 多日期混合 @2026-03-06，2026-03-10, 2026-03-15
```

### 5. 日期范围测试
```markdown
## 日期范围测试（完整格式）

- 日期范围测试任务 #任务 @L1
  - 范围完整A @2026-03-10~2026-03-12
  - 范围完整B @2026-03-10~2026-03-14 09:00:00~17:00:00

## 日期范围测试（简写格式）

- 日期范围简写任务 #任务 @L1
  - 范围简写A @2026-03-10~03-12
  - 范围简写B @2026-03-10~03-12 09:00:00~10:00:00
```

### 6. 混合模式测试
```markdown
## 混合模式测试（复杂场景）

- 混合场景测试任务 #任务 @L1
  - 混合场景A @2026-03-06 09:00:00~09:30:00, 2026-03-10~03-12 14:00:00~15:00:00, 2026-03-15 10:00:00~11:00:00
  - 混合场景B @2026-03-06, 2026-03-10~03-12, 2026-03-15 #done
  - 混合场景C @2026-03-28~2026-04-02
```

### 7. 状态标签测试
```markdown
## 带状态标签的多日期事项测试

- 状态标签测试任务 #任务 @L1
  - 状态完成A @2026-03-06, 2026-03-10, 2026-03-15 #已完成
  - 状态放弃A @2026-03-06, 2026-03-10 #已放弃
  - 状态完成B @2026-03-06, 2026-03-10 #done
  - 状态放弃B @2026-03-06, 2026-03-10 #abandoned
```

### 8. 边界测试
```markdown
## 边界测试

- 边界测试任务 #任务 @L1
  - 边界单时间 @2026-03-06 09:00:00
  - 边界时间范围 @2026-03-06 09:00:00~10:00:00
  - 边界年底 @2026-12-30, 2026-12-31, 2027-01-01
  - 边界闰年 @2024-02-28, 2024-02-29, 2024-03-01
```

### 9. 任务层级测试（保持原有结构）
```markdown
## 任务层级测试

- 层级任务A #任务 @L1
  - 层级任务B #任务 @L2
    - 层级任务C #任务 @L3
    - 层级事项A @2026-03-10 09:00:00~10:00:00
    - 层级事项B @2026-03-11 14:00:00~15:00:00
  - 层级事项C @2026-03-12
- 层级事项D @2026-03-13
```

### 10. 链接测试（保持原有结构，但添加任务）
```markdown
## 链接测试（缩进格式）

- 链接缩进测试任务 #任务 @L1
  - 链接缩进A @2026-03-10
    [链接A1](https://example.com)
  - 链接缩进B @2026-03-06, 2026-03-10
    [链接B1](https://github.com)
  - 链接缩进C @2026-03-15
    [链接C1](https://example.com/requirements)
    [链接C2](siyuan://blocks/20260220112000)

## 链接测试（不缩进格式）

- 链接不缩进测试任务 #任务 @L1
  - 链接不缩进A @2026-03-20
  [链接D1](https://example.com)
  [链接D2](https://github.com)
  - 链接不缩进B @2026-03-25, 2026-03-26
  [链接E1](https://docs.example.com)
```

### 11. 有序列表测试（保持原有结构，但添加任务）
```markdown
## 有序列表测试

1. 有序任务A #任务 @L1
   1. 有序子任务A #任务 @L2
      - 有序事项A @2026-04-01
      - 有序事项B @2026-04-02 09:00:00~10:00:00
   2. 有序子事项A @2026-04-03
2. 有序事项C @2026-04-04

## 有序列表多日期

1. 有序多日期任务 #任务 @L1
   1. 有序多日期A @2026-04-05, 2026-04-10, 2026-04-15
   2. 有序多日期B @2026-04-06~04-08 #done

## 有序列表链接测试

1. 有序链接任务 #任务 @L1
   1. 有序链接A @2026-04-20
      - [有序链接1](https://example.com)
      - [有序链接2](https://github.com)
   2. 有序链接B @2026-04-21, 2026-04-22
   - [有序链接3](https://docs.example.com)
```

## 具体修改步骤

1. 保留基础任务区（作为任务层级测试的基础）
2. 为每个测试类型添加对应的任务行
3. 将原有独立事项缩进作为子项
4. 保持事项内容和日期不变
5. 保持链接格式不变
6. 保持状态标签不变

## 预期结果

修改后的测试文件将符合 `data-format.md` 的规范：
- 每个事项都有对应的任务
- 任务层级清晰（L1/L2/L3）
- 便于测试多日期事项的迁移、拖动等功能
