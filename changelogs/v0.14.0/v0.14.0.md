# Changelog

## [0.14.0] - 2026-07-04

### Features

- **WeCom Bot**: Support WeCom bot integration for AI assistant conversations ([#78](https://github.com/MoonBottle/siyuan-plugin-bullet-journal/issues/78))
  - WeCom message sending/receiving: Chat with AI assistant through WeCom
  - Streaming message replies: Typewriter effect for streaming responses
  - Notification push: Push reminders, pomodoro events via WeCom
  - Multi-channel distribution: Notification system supports pushing to multiple channels
  - Session keepalive: WebSocket connection auto-reconnect and session persistence
- **CSS Isolation**: New scopeBytemdCss Vite plugin to isolate third-party CSS global styles, preventing conflicts with SiYuan styles

### Fixes

- **Skill Editor Styles**: Fixed dark mode skill editor rich text editor CSS conflicting with SiYuan code block font colors ([#77](https://github.com/MoonBottle/siyuan-plugin-bullet-journal/issues/77))
- **WeCom Bot**:
  - Fixed pong response misidentified as subscription response causing exponential ping growth
  - Fixed local notification push restoring WeCom session mapping from persistence
  - Fixed plugin loading syncing AI providers configuration
  - Fixed auto-connect on enable and persisting enable state with notification toggle
  - Fixed WeCom message integration with session storage for visible chat history in UI
  - Fixed status sync and single chat reply target
  - Fixed subscription response parsing to match real protocol
  - Added aibot_event_callback event callback support
  - Fixed WebSocket endpoint URL
  - Fixed missing settings wiring, ineffective notification toggle, disable not stopping monitor
  - Fixed mobile input loss, plugin null reference, notification toggle initialization
  - Fixed AI reply channel mismatch
- **Conversation Select**: Fixed ConversationListItem type incompatibility with wecom source
- **AI Chat**: Optimized wecom bot button click behavior
